# 「概念-命题-方法」三层卡片法

对每一个核心章节，制作三层知识卡片，构建完整的"是什么→为什么→怎么用"认知闭环。

> 一张好的三层卡片，不是教材的搬运工，而是**知识的重构器**。
> > 参考实例：[`函数3.md`](../math/cards/函数3.md)

---

## 卡片元数据规范

每张卡片顶部必须包含标准化 frontmatter：

```markdown
---
name: [概念名]
card_id: [序号]
chain: [所属知识链]
module: [所属模块]
type: three-layer-card
tag_type: [formal / intuitive]
grade: [高一/高二/高三]
book: [教材位置]
tags: [相关标签列表]
---
```

## 第一层：概念层（是什么）

**目标：** 建立准确、稳固的概念基础，回答"这个概念为什么会出现"和"它到底是什么"。

### 必备内容

**1. 问题的起源**

- 不要直接扔定义，先回答"为什么需要这个概念"
- 对比**旧工具解决不了什么问题**，引出**新工具的必要性**
- 用一句话概括概念的核心价值

> 例：函数概念的起源——方程只能描述静态关系，但现实世界充满动态变化（时间→位移、价格→销量），函数就是为了刻画"变化规律"而诞生的。

**2. 原始定义（教材原文）**

- 一字不差引用教材原文
- 标注出处（教材版本、页码）
- 拆解定义中的**关键词和限定词**（如"任意""唯一""确定"）

**3. 本质定义（逻辑重构版）**

- 抛开教材的集合语言，用**自己的话**重新表述
- 可配合 ASCII 图示或流程图
- 突出核心要素之间的逻辑关系

> 例：函数本质 = 输入 x → 规则 f → 输出 y，满足三条铁律（任意性、唯一性、确定性）

**4. 定义的三要素**

- 将定义拆解为可操作的具体要素
- 用表格呈现：要素 + 含义 + 核心问题 + 示例

**5. 定义的等价表述**

- 同一概念，用不同语言体系表述
- 至少覆盖：集合语言、变量语言、图像语言、程序语言
- 标注每种表述的适用场景

**6. 图像表示**

- 用文字或 ASCII 图呈现关键图像特征
- 包含判断法则（如垂直线测试）
- 标注典型图像的定义域和值域

**7. 邻近概念网络**

- 用层级图或树状图展示概念在更大知识体系中的位置
- 标注"包含于""特化于"等关系
- 说明本章概念与后续章节的接口

**8. 概念辨析**

- 选择 2-3 组最易混淆的邻近概念进行对比
- **必须用生活比喻**，避免纯数学语言堆砌
- 对比维度统一：比喻、核心动作、角色定位、本质区别

> 例：函数 vs 方程 → "机器 vs 谜语"；函数 vs 数列 → "连续录像 vs 定时拍照"

**9. 适用边界**

- 明确"什么时候成立，什么时候不成立"
- 用表格呈现：成立条件 / 反例 / 违反的铁律
- 列出定义域的隐含限制（常见陷阱清单）

## 第二层：命题层（为什么）

**目标：** 理解定理背后的逻辑链条，通过典型例题掌握定理的深层应用。

### 必备内容

**1. 核心定理**

- 每个定理包含：**精确表述** + **证明思路/逻辑链** + **前提条件**
- 关键定理标注**教材出处**
- 用表格统一呈现：定理名 + 条件 + 结论

**2. 定理链**

- 展示定理之间的逻辑依赖关系
- 用箭头图呈现：前提定理 → 当前定理 → 推论/后续定理
- 标注每个环节的前提条件

> 例：闭区间连续性 → 最值存在 → 介值定理 → 零点存在 → 二分法

**3. 定理的适用边界**

- 用表格呈现：定理 + 成立需要的条件 + 条件不满足时的反例
- 特别强调**最容易被忽略的前提条件**

**4. 典型例题的深层分析**

- 每个核心定理配 **1-3 道典型例题**
- 每道例题包含：
  - **题目**（精简呈现，保留核心结构）
  - **深层分析**（这道题在考查什么？属于什么题型？有什么隐藏条件？）
  - **完整解题过程**（关键步骤标注推理依据）
  - **方法提炼**（从这道题中提炼出的通用方法，用"→"表示逻辑链）

> **深度要求：** 不是"把答案抄一遍"，而是分析"为什么第一步要这样做""如果条件变了结论会如何""这道题和书上哪道题是同源的"。

## 第三层：方法层（怎么用）

**目标：** 建立从问题到方法的快速通路，让方法可操作、可迁移。

### 必备内容

**1. 识别信号表**

- 建立"题目关键词 → 识别信号 → 首选方法"的映射
- 用表格呈现，方便快速查阅

> 例："求定义域" → 解析式含分式/根式/对数 → 列不等式组求交集

**2. 决策树/标准操作流程**

- 每个核心方法配一个**决策树或步骤流程**
- 用 ASCII 图或编号步骤呈现
- 标注每一步的**判断依据和常见错误**

> 例：奇偶性判定三步流程，第一步标注"80%的错误来自跳过此步"

**3. 方法详解（带例题）**

- 每个方法包含：
  - **核心思想**（一句话概括这个方法的本质）
  - **标准操作流程**（分步骤，每步有具体操作）
  - **例题应用**（用具体题目演示完整操作过程）
  - **易错提醒**（这个方法最容易在哪里出错）

**4. 跨章节方法迁移**

- 用表格呈现：原领域 + 核心方法 + 迁移目标 + 迁移实例
- 每个迁移方向至少给一个**具体公式/例题**

> 例：函数单调性的差值比较法 → 迁移到数列单调性 → 比较 aₙ₊₁ − aₙ 与 0

**5. 典型题型模板**

- 对高频题型，提炼**可复用的解题模板**
- 模板包含：题目特征 → Step 1/2/3 → 注意事项
- 用代码块或步骤列表呈现

## 补充模块

### 常见误区与纠正

- 至少列出 5-8 个最常见误区
- 每个误区包含：误区表述 + 错误根源 + 正确理解
- 用表格呈现，便于快速扫描

### 知识坐标定位

- 用三维坐标系定位本概念：
  - **X 轴（对象维度）**：本章研究什么对象？
  - **Y 轴（结构维度）**：在知识结构的哪一层？
  - **Z 轴（能力维度）**：需要调动什么核心能力？

## 写作原则

1. **先写起源，再写定义** —— 不要让读者直接面对抽象符号
2. **多用比喻，少用术语** —— "机器 vs 谜语"比"映射的特殊形式"好记
3. **每个表格都有目的** —— 对比、梳理、速查，不要堆砌信息
4. **例题必须带提炼** —— 做完题要回答"从这道题学到了什么通用方法"
5. **标注出处和前提** —— 定理标注教材页码，方法标注适用条件
6. **ASCII 图辅助理解** —— 图像、流程、网络用文字图呈现

## 常见问题

**Q: 三层卡片要写多详细？**
A: 概念层最详（起源+本质+辨析+边界），命题层中等（定理+例题分析），方法层最实用（流程+例题+迁移）。总字数控制在 2000-4000 字，确保一张卡片能在 15 分钟内读完。

**Q: 已有错题本，还需要三层卡片吗？**
A: 错题本解决"这一题会不会"，三层卡片解决"这一类题怎么想"。两者互补，优先保证三层卡片覆盖核心章节。

**Q: 什么算"核心章节"？**
A: 考试高频考点：函数概念、导数、三角函数、平面向量、圆锥曲线、数列、概率统计。一张卡片对应一个核心概念（如"函数"），不是对应一整章。

**Q: intuitive 标签的概念怎么写卡片？**
A: 与 formal 卡片结构相同，但概念层更侧重直观理解（生活实例、几何直觉），命题层侧重定性分析而非严格证明，方法层侧重思维迁移而非公式套用。
