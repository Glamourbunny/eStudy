## 🛠️ 体系结构总结：从连续微积分到离散 Python 代码的转化跨越

在本体系中，核心在于理清从 **连续方程（Continuous Equations）** 向 **离散代码（Iterative Code）** 落地的计算思维链条。以 **[Sci-Link/Python]** 的天体轨道和阻力抛体运动数值仿真为例，其内在的逻辑映射如下：

1. **物理现实与连续建模**：
   物体在空间受到连续变化的力（如引力 $F = G\frac{Mm}{r^2}$ 或阻力 $f = kv^2$），根据牛顿第二定律建立连续微分方程：
   $$\frac{dv}{dt} = a(x, v), \quad \frac{dx}{dt} = v$$

2. **考场极限（算力爆炸）**：
   在引入真实空气阻力后，上述方程在数学上表现为非线性微分方程，**无法求出解析显式解（即无法写出符合高考要求的 $x(t)=\dots$ 完美代数公式）**。人类大脑算力在此处卡死。

3. **算法思维与离散化（工程妥协）**：
   利用教材中微元法的逆向思维，将无限小的连续微分 $dt$ 替换为计算机中可执行的**有限小离散时间步长（Discrete Time Step）** `dt = 0.001`。此时，连续的变化率被转化为离散的**差分**。

4. **代码迭代落地**：
   利用**迭代代码（Iterative Code）**的循环结构，在每一帧中无脑执行状态转移：
   ```python
   # 每一个 dt 时间步长内的离散状态转移
   a = F_total(x, v) / m       # 计算当前加速度（代数几何化）
   v = v + a * dt              # 速度离散步进（差分递推）
   x = x + v * dt              # 位移离散步进（数值积分）
