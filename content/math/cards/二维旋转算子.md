# 二维旋转算子

> **Intuitive 卡片不教你"怎么算"，而是教你"算什么"。** "旋转算子"这个词高中课本不提，但复数乘法的几何意义——"乘 i 就是转 90°"——正是旋转算子的雏形。这张卡片让你认清：凡是涉及"转一个角度"的题，复数法是隐藏的王牌。

---

## 视角层（像什么）

### 核心隐喻：时钟指针的拨动

**二维旋转算子 = 你有三根指针——秒针、分针、时针——每根的位置都是一个向量。你要转动某一根指针，只需要一个"转动指令"**

```
  当前位置                      转动后位置
      12                             12
      │                              │
   9 ─┼─ 3  ── 逆时针转 90° ──→  9 ─┼─ 3
      │            ×i                │
      6         →→→→→→               6
      时针指向3                      时针指向12
      (3 点钟)                       (12 点钟)
```

**旋转指令的三条性质：**

| 性质 | 时钟场景 | 复数语言 |
|------|---------|---------|
| **乘以 i = 逆时针转 90°** | 从 3 点拨到 12 点 | 乘 i：逆时针转 π/2 |
| **叠加旋转 = 角度相加** | 先转 30°再转 45° = 总共转 75° | 乘 (cos30°+isin30°) 再乘 (cos45°+isin45°) = 乘 (cos75°+isin75°) |
| **旋转不改变长度** | 指针长度不变，只改变指向 | \|旋转后的向量\| = \|原向量\| |

> **一句话直觉：** 旋转算子 = 时钟的发条——它只转动方向，不改变指针的长度。

### 动态想象：万花筒里的旋转

把平面上的点想象成万花筒里的彩色碎片：

```
  初始碎片 P(x,y)                              旋转后 P'(x',y')
       ●                                          ●
        \                                        /
         \         旋转 θ 角                    /
          \       ──────────→                  /
           \                                  /
            O    转轴（原点）                  O
```

**旋转算子的"配方"：**
```
  P' = P 绕原点逆时针转 θ
  
  x' = x·cosθ - y·sinθ
  y' = x·sinθ + y·cosθ
  
  等价于：乘以复数 (cosθ + i·sinθ)
```

**高考里的翻版：**
- "将点 A 绕原点逆时针旋转 60°，求新坐标"——直接乘复数，比坐标旋转公式快得多
- "正三角形 ABC，A(0,0), B(2,0)，求 C"——B 绕 A 转 60° 即得 C
- "四边形 ABCD 是正方形，求 D 点坐标"——相邻顶点之间差了 90° 旋转

### 反直觉警示：大脑会在哪里翻车

| 翻车点 | 直觉告诉你 | 实际真相 |
|--------|-----------|---------|
| **旋转 90° ≠ 换个坐标** | "转 90° 就是把 (x, y) 变成 (y, x)" | (x, y) 逆时针转 90° → (-y, x)，不是 (y, x)！(y, x) 是关于 y=x 对称 |
| **复数乘法 ≠ 一般的乘法** | "两个点乘起来，我应该得到更大/更小的点" | 两个复数相乘，模长相乘但辐角相加——结果的重点是"转了多大角度" |
| **旋转中心不一定是原点** | "转就行" | 绕非原点旋转 = 先平移到原点，旋转，再平移回去——三步操作 |

## 联结层（连什么）

### 跨学科锚点

**1. 物理：力矩扳手的旋转**

```
  扳手把手从水平位置（向量 r）旋转到某个角度
  
  r 绕转轴旋转 θ = r × (cosθ + i·sinθ)
  
  每一次拧螺丝的动作，都是旋转算子在物理世界的投影
```

**2. 计算机：游戏角色的转身**

```python
# 游戏中角色面向角度调整
facing_angle += 30  # 右转 30 度
forward_vector = rotate(forward_vector, 30°)

# 实际上就是复数乘法：
# new_forward = forward × complex(cos30°, sin30°)
```

每一帧游戏画面里的转身动画，底层都在调用旋转算子。

**3. 生活：旋转楼梯**

```
  螺旋上升时，水平投影每一阶都在旋转一个固定角度
  
  第 n 阶的水平位置 = 第 1 阶 × (旋转算子)^(n-1)
```

旋转楼梯就是旋转算子的多次迭代。

### 向 Formal 的桥梁

**拆包后，落回哪条 formal 定理？**

| 高考场景 | 拆包结论 | Formal 着陆点 |
|---------|---------|--------------|
| 点绕原点旋转 | 这是"转钟"操作 | 复数三角形式：乘 (cosθ+isinθ) |
| 正多边形顶点坐标 | 每次旋转相同角度 | 单位根：zₖ = e^(i·2πk/n) |
| 向量的几何变换 | 绕某点旋转 | 平移到原点 → 旋转 → 平移回去 |
| 三角恒等式推导 | 旋转叠加 | cos(α+β) = cosαcosβ-sinαsinβ 来自复数乘法 |

**首选工具**：`复数乘法`——把坐标变成 x+yi，乘 cosθ+isinθ，再拆回坐标
**次选工具**：旋转矩阵——[[cosθ, -sinθ], [sinθ, cosθ]]，本质一样但书写更繁

              └────────┬────────┘
                       │

              ┌─────────────────┐
              │ 二维旋转算子    │  ← 核心：乘法 = 旋转

                       │
         ┌─────────────┼─────────────┐
         │             │             │

**拆包调用优先级：**
1. 看到"旋转""绕原点转 θ 角" → 复数乘法，乘 cosθ+isinθ
2. 看到"正三角形""正方形""正 n 边形" → 单位根方法
3. 看到"绕非原点旋转" → 三步走：平移→旋转→平移回

### 历史/哲学视角

**旋转算子的发现，让数学家第一次用"乘法"来描述"旋转"。** 在复数出现之前，描述平面旋转需要两个方程（x' 和 y' 的表达式）和一个矩阵。但复数用一个数就同时编码了"模长不变"和"角度叠加"两个信息。

这就是为什么物理学家和工程师几乎离不开复数：**用复数描述二维旋转，比用坐标+三角函数简洁一个数量级。** 1800 年韦塞尔（Wessel）和 1806 年阿尔冈（Argand）各自独立发现了复数的几何表示，从此"乘法=旋转"的直觉流传至今。

## 迁移层（何时调用）

### 三类场景判定表

| 场景 | 识别信号 | 不拆包的代价 | 拆包收益 |
|:---|:---|:---|:---|
| 🎭 **包装** | "绕原点旋转""顺时针转 θ 角""点 P 经旋转后变为 P'" | 用坐标旋转公式硬代，四组三角函数容易出错 | 复数法：一行完成旋转 |
| 🔀 **嵌套** | 多步旋转叠加："先转 α 再转 β 再转 γ" | 坐标法每步都要展开，三阶嵌套 sin/cos 堆成山 | 复数连乘：角度直接相加 |
| 🆕 **新定义** | "定义平面上一个旋转变换 T_θ: P→P'" | 被"变换"字样吓住 | 认出就是旋转算子，直接复数法 |

### 拆包方向标（10 秒流程）

```
Step 1：识别结构类型（3 秒）
  • 看到"旋转"+"角度" → 时钟模型，拆"旋转包装"
  • 看到"正多边形顶点坐标" → 等角度旋转，拆"单位根包装"

Step 2：剥外壳找骨骼（4 秒）
  • 将点坐标 (x, y) 变成复数 z = x+yi
  • 旋转算子 = cosθ + i·sinθ（逆时针），或 cosθ - i·sinθ（顺时针）
  • 新点 = z × 旋转算子

Step 3：锁定 Formal 着陆点（3 秒）
  • 写下：拆包→【复数三角形式乘法】，立即切换严格语言
```

### 向 Formal 切换铁律

> **拆包完成后，30 秒内必须切换为 formal 语言。**

| 场景 | 拆包结束的标志 | 切换后的 formal 动作 |
|:---|:---|:---|
| 包装题 | 认出"旋转 = 复数乘 (cosθ+isinθ)" | 写出 z' = z·(cosθ+isinθ)，即 (x+yi)(cosθ+isinθ)，分离实虚部 |
| 嵌套题 | 确认每次旋转的角度 | 连乘：z_end = z_start × ∏(cosθₖ+isinθₖ)，实虚部分离 |
| 新定义题 | 认出"变换 = 旋转" | 用复数语言替换自定义符号，直接代数运算 |

**违反此铁律的后果**：用"转钟"的直觉转了方向但忘了 cos/sin 具体数值，导致角度符号弄反。

### 直觉陷阱清单（仅拆包场景）

| 陷阱 | 何时失效 | 纠错方式 |
|------|---------|---------|
| **顺时针/逆时针搞反** | 题目说"顺时针转"但写了 +isinθ | 顺时针转 = 乘 (cosθ - i·sinθ)，负号在 sin 前面 |
| **绕非原点旋转直接乘** | 旋转中心不是原点时，直接复数乘 | 必须三步：z→z-z₀→(z-z₀)·旋转算子→+z₀ |
| **忘记复数乘法有实虚部运算** | 乘完旋转算子后忘了拆成 x'+y'i | (a+bi)(c+di) = (ac-bd)+(ad+bc)i，必须分离实虚部 |

## 一句话总结

> **二维旋转算子 = 时钟的拨动齿轮——它只转指针的方向，不变指针的长度。用复数描述旋转，比坐标旋转公式快一倍：乘一个复数就等于转一个角。正 n 边形、绕点旋转、三角恒等式推导，都是它的主场。**
