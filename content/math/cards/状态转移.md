# 状态转移

> **Intuitive 卡片不教你"怎么算"，而是教你"算什么"。** 一个系统有几个状态，下一步可能切换到另一个状态——这种"跳来跳去"的结构，就是状态转移。高考里常见的马尔可夫链、递推概率、递推计数，本质都是"状态转移"换了不同的马甲。

---

## 视角层（像什么）

### 核心隐喻：接力赛——每次交接棒，状态随之改变

**状态转移 = 一场多棒接力赛——每一棒对应一个状态，交棒规则决定了下一棒的方向**

```
  第一棒                       第二棒                       第三棒
┌────────┐  交棒规则  ┌────────┐  交棒规则  ┌────────┐
│ 状态 A │ ──→ 60% → │ 状态 B │ ──→ 40% → │ 状态 C │
│        │ ──→ 40% → │ 状态 C │           │        │
└────────┘           └────────┘           └────────┘
                          │  ──→ 60% → ┌────────┐
                          │            │ 状态 A │
                          └────────────┘        ┘
```

**接力赛思维的三条铁律：**

| 铁律 | 接力赛场景 | 违反的后果 |
|------|-----------|-----------|
| **状态穷举**：所有可能的状态必须列全 | 赛道上有 A/B/C 三支队伍 | 漏了某个状态，转移规则不闭合 |
| **概率/数量归一**：所有转移方向的总和 = 全部 | 三支队伍各自的交棒去向加起来必须是 100% | 转移概率之和不为 1（或数量不守恒） |
| **递推继承**：下一步状态分布 = 当前分布 × 转移规则 | 第 n 棒的交棒状态 = 第 n-1 棒状态 × 交棒规则 | 试图一步算出很远以后的状态分布 |

> **一句话直觉：** 状态转移就是"系统在几个状态之间跳来跳去，跳的规则是固定的"——先列出所有状态，再写出转移规则（概率/数量），然后从初始状态开始一步步推演。

### 动态想象：天气的马尔可夫链

```
  天气只有两种状态：晴（S）和雨（R）

  转移规则（概率）：
  • 今天晴 → 明天晴的概率 70%，雨的概率 30%
  • 今天雨 → 明天晴的概率 40%，雨的概率 60%

  状态转移图：

      0.7          0.4
    ┌──→ (晴) ←──┐
    │   ↗  ↘    │
   0.3 │    │   0.6
    │  ↙    ↖   │
    └─ (雨) ────┘
      状态间按概率跳转

  转移矩阵：P = [0.7  0.3]
                 [0.4  0.6]

  今天晴：分布 = [1, 0]
  明天：   [1, 0] × P = [0.7, 0.3]
  后天：   [0.7, 0.3] × P = [0.61, 0.39]
  ...
```

### 反直觉警示

| 翻车点 | 直觉告诉你 | 实际真相 |
|--------|-----------|---------|
| **终结状态存在** | "一直跳下去总会停在某个状态" | 很多马尔可夫链没有终结状态，永远在跳——但会趋近稳态分布 |
| **一步就够了** | "算一步转移概率就可以推所有" | 多步转移需要矩阵乘幂，不是简单叠加 |
| **状态必须是对称的** | "A→B 的概率应该等于 B→A" | 完全不必——今天晴明天雨 30%，今天雨明天晴 40%，不对称很正常 |

## 联结层（连什么）

### 跨学科锚点

**1. 生物学：基因频率的世代转移（哈迪-温伯格）**
```
种群中基因型 AA, Aa, aa
每代随机交配 → 基因频率按固定规则转移
初始频率 → 第一代 → 第二代 → … → 平衡状态
这就是状态转移！状态 = 基因频率，转移 = 随机交配规则
```

**2. 计算机科学：自动机（状态机）**
```
电梯的状态转移：
  状态：静止/上升/下降/开门/关门
  转移：按楼层按钮 → 关门 → 上升/下降 → 到达 → 开门
任何一个计算机程序都可以用"状态+转移规则"来描述
```

**3. 棋类游戏：棋盘上的状态转移**
```
围棋：每一步落子改变了棋盘的状态
当前棋盘状态 → 落子规则 → 下一棋盘状态
AI 下棋就是在"状态空间"中搜索最佳转移路径
```

```
  系统有多个状态
      │
```

## 迁移层（何时调用）

### 三类场景判定表

| 场景 | 识别信号 | 拆包收益 |
|:---|:---|:---|
| 🎭 **包装** | "每轮…""每步以概率 p…"藏在概率/计数题中 | 识别为状态转移，立刻画状态图 |
| 🔀 **嵌套** | 多阶段决策，每个阶段有不同转移规则 | 拆成多段状态转移，每段独立设转移矩阵 |
| 🆕 **新定义** | "定义系统在时刻 n 的概率分布为…" | 翻译成状态转移的语言 |

### 拆包方向标（10 秒流程）

```
Step 1：穷举状态（3 秒）
  • 系统有哪些互斥的状态？
  • 每个状态用符号表示（A/B/C 或 S1/S2/S3）

Step 2：画状态转移图（4 秒）
  • 箭头：A→B 表示"从 A 可以转到 B"
  • 标注转移概率/数量规则

Step 3：锁定 Formal 着陆点（3 秒）
  • 写转移矩阵或递推公式
  • 用矩阵乘法（概率）或数列方法（数量）求解
```

### 直觉陷阱清单

| 陷阱 | 何时失效 | 纠错方式 |
|------|---------|---------|
| **状态遗漏** | 系统有些状态隐藏在条件中 | 读完题先穷举所有可能状态，不要遗漏 |
| **概率不归一** | 所有转移概率之和 ≠ 1 | 检查每个状态的所有"出口"概率之和是否为 1 |
| **混淆状态和步骤** | 第 n 步的状态 vs 第 n 步的转移 | 明确：状态是"在哪儿"，转移是"怎么走" |
| **稳态误判** | 算了两步就以为到了稳定状态 | 验证：状态分布不再随步数变化时才是稳态 |

---

## 一句话总结

> **状态转移 = 系统在有限个状态间按规则跳转——穷举所有状态，画出转移图，写出转移规则（矩阵或递推公式），然后从初始状态出发一步步推演。概率问题用转移矩阵，计数问题用递推公式，本质都是"此刻在哪儿 → 下一步去哪儿"。**
