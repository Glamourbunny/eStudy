# 算法思维

> **Intuitive 卡片不教你"怎么算"，而是教你"算什么"。** 空间向量最强大的地方不是"能算"，而是"能用固定流程批量处理各种几何问题"。这张卡片让你看到：空间向量本质上是一套"自动化几何解题机"。

---

## 视角层（像什么）

### 核心隐喻：快递分拣中心的机器臂

**算法思维 = 机器臂按照固定程序，自动分拣不同形状的包裹**

```
  包裹进入 ──→ 传感器扫描 ──→ 电脑计算坐标 ──→ 机器臂抓取 ──→ 放入对应货架
  
  ┌──────────┐     ┌──────────┐     ┌──────────┐     ┌──────────┐
  │ 问题输入   │ →  │ 建立坐标系│ →  │ 公式运算  │ →  │ 输出结果  │
  │ "求异面    │     │ 设点坐标  │     │ 向量法    │     │ "距离=2"  │
  │  直线距离" │     │          │     │ 运算     │     │          │
  └──────────┘     └──────────┘     └──────────┘     └──────────┘
  
  不管包裹长什么样，机器臂都用同一套程序处理
```

**机器臂的固定程序（空间向量解题五步法）：**

| 步骤 | 分拣场景 | 空间向量解题 |
|------|---------|------------|
| **1. 建系** | 传感器扫描，确定包裹位置 | 建立空间直角坐标系，写出关键点坐标 |
| **2. 写向量** | 计算抓取路径 | 用终点-起点写出需要的方向向量 |
| **3. 选工具** | 选择对应抓爪 | 根据问题类型选向量工具（数量积/法向量/投影） |
| **4. 套公式** | 机器臂按程序移动 | 代入公式计算 |
| **5. 出结果** | 包裹放入货架 | 得出距离/角度/判定结果 |

> **一句话直觉：** 算法思维 = 把几何问题丢进"坐标工厂"的传送带——进去的是几何图形，出来的是数字答案。中间过程是标准化的，不需要每次"另想办法"。

### 动态想象：流水线上的自动质检

```
  工厂流水线上每个产品经过三个质检站：
  
  质检站 1：位置关系判定
    → 平行？相交？异面？（方向向量 vs 法向量）
  
  质检站 2：度量计算
    → 距离？角度？（投影公式 vs 夹角公式）
  
  质检站 3：几何判定
    → 共线？共面？垂直？（向量线性关系）
  
  每个产品都经过同样的检查流程——这就是算法思维的威力
```

**高考里的翻版：**
- "求异面直线距离" —— 传统几何法需要找公垂线，算法思维：建系→写向量→套异面直线距离公式
- "求二面角" —— 传统法需要找棱、作垂线，算法思维：两平面法向量夹角
- "证明线面垂直" —— 传统法需要找两条相交直线分别垂直，算法思维：方向向量 ∥ 法向量

### 反直觉警示：大脑会在哪里翻车

| 翻车点 | 直觉告诉你 | 实际真相 |
|--------|-----------|---------|
| **建系不是越漂亮越好** | "我要尽量找到对称的坐标系" | 选系的标准是"点的坐标好写"，不对称也可能比对称好用 |
| **向量法和几何法不是对立的** | "用向量就是逃避几何思考" | 考场上先算法思维（建系+套公式），再用几何思维验证答案合理性 |
| **不是所有题都适合建系** | "无所谓，建系就行" | 如果已知条件全是角度和长度关系（没给坐标），几何法可能更快 |

## 联结层（连什么）

### 跨学科锚点

**1. 计算机：线性代数求解器**

```python
# 求两个三维向量的夹角
import numpy as np

a = np.array([1, 2, 3])
b = np.array([4, 5, 6])
cos_theta = np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))
theta = np.arccos(cos_theta)
```

空间向量解题的本质就是手动执行计算机的线性代数计算——一样的流程、一样的公式。

**2. 生活：GPS 导航**

```
  你的位置 = (经度, 纬度, 海拔) —— 三维坐标
  目的地位置 = 另一个三维坐标
  
  最短距离 = |目的地向量 - 你的位置向量| —— 空间向量模长公式
```

GPS 导航的底层逻辑就是空间向量运算的连续版本。

**3. 物理：无人机路径规划**

```
  无人机起点 A(x₁, y₁, z₁)，目标 B(x₂, y₂, z₂)
  需避开的障碍物 = 某些坐标区域
  
  路径 = A→B 的方向向量，障碍物到路径的距离 = 点到直线距离公式
```

### 向 Formal 的桥梁

**拆包后，落回哪条 formal 定理？**

| 高考场景 | 拆包结论 | Formal 着陆点 |
|---------|---------|--------------|
| 求距离（点面/线线/异面） | 识别"距离工厂"第几号程序 | 点面距离公式、异面直线距离公式 |
| 求角度（线线/线面/二面角） | 识别"角度工厂"第几号程序 | 向量夹角公式：cosθ=\|a·b\|/(\|a\|\|b\|) |
| 判定位置关系 | 识别"关系判定器" | 方向向量关系、法向量关系 |
| 证明题 | 把几何命题翻译成向量等式 | 向量共线/共面/垂直的代数条件 |

**首选工具**：`坐标运算`——所有问题先建系，写出坐标，然后机械化运算
**次选工具**：几何法——当坐标不易写时，退回到传统的几何推理

### 历史/哲学视角

**算法思维的兴起标志着几何从"艺术"变成了"工程"。** 传统综合几何（欧几里得风格）更像一门艺术——每道题需要独特的辅助线灵感。而笛卡尔（1637）引入坐标系后，几何开始"算法化"：建系→代数运算→得出结论。

到了 20 世纪计算机出现后，这种思维被发挥到极致——计算机不能"灵光一闪"，只能按程序计算。**空间向量解题法就是一台"手动几何计算机"**，它不美，但可靠、稳定、不易失分。

## 迁移层（何时调用）

### 三类场景判定表

| 场景 | 识别信号 | 不拆包的代价 | 拆包收益 |
|:---|:---|:---|:---|
| 🎭 **包装** | 复杂立体几何题，传统辅助线难找 | 辅助线画一堆，理不清关系 | 建系→写坐标→套公式，10 分钟变 5 分钟 |
| 🔀 **嵌套** | 多步几何关系：先证平行再求距离再求体积 | 每步换方法，思路断裂 | 一套坐标体系贯穿所有步骤，前后结果可复用 |
| 🆕 **新定义** | 较少以新定义出现 | — | — |

### 拆包方向标（10 秒流程）

```
Step 1：识别结构类型（3 秒）
  • 看到立体几何大题 → 启动"算法思维"模式
  • 检查：已知条件有坐标或容易建系？→ 向量法优先

Step 2：启动标准化流程（4 秒）
  • 建系 → 写点坐标 → 写向量 → 选公式 → 代入 → 输出
  
Step 3：锁定 Formal 着陆点（3 秒）
  • 写下：拆包→【空间向量坐标运算】，立即启动五步流程
```

### 向 Formal 切换铁律

> **拆包完成后，30 秒内必须切换为 formal 语言。**

| 场景 | 拆包结束的标志 | 切换后的 formal 动作 |
|:---|:---|:---|
| 包装题 | 确认"这道题适合建系" | 写下坐标系，逐点赋值，严格遵守向量运算规则 |
| 嵌套题 | 确认多步间可共享坐标 | 每步独立计算，但坐标共用，最后一步汇总 |

**违反此铁律的后果**：建系后乱写向量（方向搞反、模长算错），导致整题崩盘。

---

### 直觉陷阱清单（仅拆包场景）

| 陷阱 | 何时失效 | 纠错方式 |
|------|---------|---------|
| **坐标系选错** | 选了难写坐标的建系方式 | 优先选"坐标中多含 0 或相等值"的建系方式 |
| **向量方向写反** | AB 写成 B-A 还是 A-B 搞混 | 方向向量 = 终点 - 起点，确定后全文一致 |
| **法向量忘记"任意性"** | 死磕法向量必须是什么值 | 法向量方向不唯一，只要跟平面内两个不共线向量都垂直即可 |

## 一句话总结

> **算法思维 = 把立体几何变成"坐标工厂"里的标准化流水线——输入图形，经过建系→写坐标→套公式→输出结果。核心优势不是"算得快"，而是"不用每次另想办法"，以及"一套坐标吃遍所有小题"。**
