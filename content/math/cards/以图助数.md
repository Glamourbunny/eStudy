# 以图助数

> **Intuitive 卡片不教你"怎么算"，而是教你"算什么"。** 代数推了十行还没头绪？画一张草图——它不给你答案，但给你方向。

---

## 视角层（像什么）

### 核心隐喻：地图导航——在代数迷宫中快速定位

**以图助数 = 在代数计算的迷宫里打开一张地图**

```
  代数迷宫（纯符号）          地图模式（有图辅助）
┌────────────────────┐    ┌────────────────────┐
│  解 x²-5x+6 > 0   │    │  抛物线开口向上     │
│  分类讨论…         │    │  与 x 轴交于 2 和 3 │
│  情况1…情况2…     │    │  → x < 2 或 x > 3  │
│  （30 秒）         │    │  （5 秒）           │
└────────────────────┘    └────────────────────┘

  没有地图，凭感觉走          打开地图，路一目了然
```

**地图导航的三条铁律：**

| 铁律 | 场景 | 违反的后果 |
|------|------|-----------|
| **先看全貌**：画图先看大趋势 | 抛物线开口方向、增减趋势、零点位置 | 上来就分类讨论，不知该分几段 |
| **标注关键点**：零点、极值点、交点 | 标出后解题方向由这些点决定 | 算了一堆，漏了关键转折 |
| **草图就够**：不需要精准，只需要结构正确 | 凹口向上还是向下？交 x 轴在哪？ | 花时间画精确图，不如草图画完赶紧算 |

> **一句话直觉：** 代数算不清楚的时候，画一张草图——标出零点、极值、交点，这些关键点会告诉你分类讨论的区间。

### 动态想象：画图三笔法

```
第一笔：画走势（开口向哪？单调性？）
  y = 2x² - 8x + 6
  → 开口向上（系数 2>0），先画一个 U

第二笔：标关键点（零点？极值点？交点？）
  2x²-8x+6 = 0 → x=1, x=3（零点）
  对称轴 x=2，顶点 (2,-2)（最低点）

第三笔：分区间（关键点把数轴切成几段？）
  x<1 | 1<x<3 | x>3
  → 每段取一个测试值，验证函数值的正负
```

**对比纯代数：**
- 纯代数：解不等式 2x²-8x+6>0 → 分解因式 (2x-2)(x-3)>0 → 分类讨论两个括号的正负
- 画图法：U 形在 x<1 和 x>3 时在 x 轴上方 → 直接出答案

### 反直觉警示

| 翻车点 | 直觉告诉你 | 实际真相 |
|--------|-----------|---------|
| **画图太慢** | "画图耽误时间，直接算更快" | 画草图的 20 秒可以省掉分类讨论的 2 分钟 |
| **图不精确不敢用** | "图画不准，算了还是列式吧" | 草图的精度够你做定性判断（正负、单调、交点个数），定量计算在后面 |
| **图不能替代证明** | "图画出来就是这样，还需要证？" | 图给方向，代数给判决——两者缺一不可 |

## 联结层（连什么）

### 跨学科锚点

**1. 股市分析：K 线图助判趋势**
```
股价数字（代数）→ K 线图（图形）
"当前价格 10.5，昨收 10.2，最高 10.8，最低 9.9"
↑ 四个数字不好理解走势
↓
一根 K 线柱：一眼看到涨跌、振幅
```

**2. 气象：雷达图助判降雨**
```
各站点降雨量（数值表格）→ 雷达回波图（可视化）
不看 100 个数字，看一张彩色图
```

### 向 Formal 的桥梁

| 代数问题 | 画什么图 | 图告诉你 | Formal 着陆点 |
|---------|---------|---------|--------------|
| 解不等式 f(x)>0 | f(x) 的草图 | 哪些区间在 x 轴上方 | 不等式.md |
| 讨论方程 f(x)=0 的实根个数 | f(x) 图像与 x 轴的交点个数 | 几个交点 = 几个根 | 函数.md, 零点定理 |
| 比较 f(x) 和 g(x) 的大小 | 两条曲线在同一坐标系 | 谁在上谁就大 | 函数.md |
| 求参数范围使 f(x)>0 恒成立 | 图像全部在 x 轴上方 | 最小值要 >0 | 不等式.md, 含参问题.md |

### 概念网络坐标

```
  代数问题 ──画草图──→ 图形直觉（方向/范围）
      │                    │
      ▼                    ▼
  纯代数推导           图形导航 + 代数验证
  （容易迷失）           （快速准确）
```

## 迁移层（何时调用）

### 三类场景判定表

| 场景 | 识别信号 | 拆包收益 |
|:---|:---|:---|
| 🎭 **包装** | 不等式穿了"函数值范围"的外衣 | 画图看图像的上下位置，秒出区间 |
| 🔀 **嵌套** | 参数+不等式+恒成立三合一 | 图定参数范围，代数算精确边界 |

### 拆包方向标（10 秒流程）

```
Step 1：判断是否需要画图（3 秒）
  • 问的是"范围/个数/位置关系"？→ 画
  • 问的是"精确数值"？→ 直接算

Step 2：三笔法画草图（4 秒）
  • 走势 → 关键点 → 分区间

Step 3：锁定 Formal 着陆点（3 秒）
  • 图给区间，代数给精确边界点
```

---

### 直觉陷阱清单

| 陷阱 | 何时失效 | 纠错方式 |
|------|---------|---------|
| **图替代证明** | 用"图上看起来"做最终结论 | 图给方向，代数才是答案 |
| **忘画渐近线** | 有分母的函数的渐近行为被忽略 | 分式函数必须画渐近线 |
| **只看局部** | 只画了零点附近，远处走势出错 | 草图虽糙，整体趋势必须对 |

## 一句话总结

> **以图助数是代数的 GPS——画一张草图，零点/极值/交点帮你切分数轴，把"海量分类讨论"变成"看图选区间"。图给方向，代数给精度。**
