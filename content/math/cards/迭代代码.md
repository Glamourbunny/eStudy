# 迭代代码

> **Intuitive 卡片不教你"怎么算"，而是教你"算什么"。** 你不需要会写代码——但要能在脑子里"模拟"代码的执行：循环跑了几次？每次循环里变量怎么变？什么时候退出循环？这在高考的流程图/算法题中是核心能力。

---

## 视角层（像什么）

### 核心隐喻：自动流水线——同样的操作反复执行，每次输入和输出首尾相接

**迭代代码 = 工厂的自动流水线——机器臂对每个经过的产品执行同样的操作，上一个产品的结果可能影响下一个**

```
  原料进入                       成品输出
    │                              ▲
    ▼                              │
┌─────────────────────────────────────┐
│  ┌───┐   ┌───┐   ┌───┐   ┌───┐    │
│  │操作│→  │操作│→  │操作│→  │操作│   │
│  │ 1  │   │ 2  │   │ 3  │   │ N  │   │
│  └───┘   └───┘   └───┘   └───┘    │
│    流水线：同样的操作，不同的产品    │
│    每经过一道工序，产品离成品更近一步 │
└─────────────────────────────────────┘
```

**流水线思维的三条铁律：**

| 铁律 | 流水线场景 | 违反的后果 |
|------|-----------|-----------|
| **循环体**：每次循环执行的是同一段逻辑 | 每过一道工序执行同样的打磨动作 | 搞不清循环里到底做了什么 |
| **状态更新**：每次循环后某些变量的值改变了 | 每过一道工序产品变得更光滑 | 忘记更新关键变量，陷入死循环 |
| **退出条件**：满足某个条件时必须停下 | 产品表面光洁度达标 → 停止流水线 | 多跑或少跑，结果错误 |

> **一句话直觉：** 迭代代码就是"把一段操作重复执行，每次基于上次的结果做微调"——你的大脑要能模拟：循环跑了几次、每轮变量值是多少、什么时候该停。

### 动态想象：在脑子里跑一遍循环

```
  s = 0
  for i = 1 to 5:
      s = s + i*i

  大脑模拟执行：

  第 1 轮：i=1, s = 0 + 1 = 1
  第 2 轮：i=2, s = 1 + 4 = 5
  第 3 轮：i=3, s = 5 + 9 = 14
  第 4 轮：i=4, s = 14 + 16 = 30
  第 5 轮：i=5, s = 30 + 25 = 55
  循环结束（i=5 之后不再满足 i≤5 的条件）

  最终 s = 55
```

### 反直觉警示

| 翻车点 | 直觉告诉你 | 实际真相 |
|--------|-----------|---------|
| **循环跑了 N 次** | "for i=1 to N 当然跑 N 次" | 小心 N=0 时一次不跑；小心 break/continue 改变次数 |
| **顺序无关** | "循环内先算 a 还是先算 b 都一样" | 顺序可能影响结果——如果 b 依赖新的 a 值 |
| **退出条件就是循环条件的反** | "i≤N 的相反是 i>N" | 边界值（N=0, N=1）经常是陷阱 |

## 联结层（连什么）

### 跨学科锚点

**1. 物理实验：逐次测量取平均**
```
测量重力加速度 g：
  第 1 次：g₁ = 9.78
  第 2 次：g₂ = 9.82
  ...
  第 N 次：g_N = 9.80

  每个测量值存下来，最后取平均
  循环体：做一次实验 + 记录数据
  循环次数：N 次
```

**2. 音乐：节拍器循环**
```
节拍器每拍执行一次：
  如果鼓点在拍子上 → 敲鼓
  否则 → 跳过

  循环体：判断 + 动作
  循环次数：直到曲终
```

## 迁移层（何时调用）

### 三类场景判定表

| 场景 | 识别信号 | 拆包收益 |
|:---|:---|:---|
| 🎭 **包装** | 题目给出流程图或伪代码 | 大脑模拟执行循环，逐轮跟踪变量 |
| 🔀 **嵌套** | 循环里套条件判断或另一个循环 | 从最内层往外逐层模拟 |
| 🆕 **新定义** | "定义操作：对序列反复执行…" | 识别为循环操作，确定循环体和终止条件 |

### 拆包方向标（10 秒流程）

```
Step 1：识别循环结构（3 秒）
  • 是 for（固定次数）还是 while（条件终止）？
  • 循环变量从几开始？到几结束？步长是几？

Step 2：模拟执行前 2-3 轮（4 秒）
  • 写出第 1、2、3 轮后各变量的值
  • 找规律：能不能写出第 k 轮的通式？

Step 3：锁定 Formal 着陆点（3 秒）
  • 找到通式 → 直接用公式算最终结果
  • 找不到通式 → 手动模拟到循环结束
```

### 直觉陷阱清单

| 陷阱 | 何时失效 | 纠错方式 |
|------|---------|---------|
| **边界值判断错误** | 条件 i<N 和 i≤N 差一轮 | 代入 N=1 验证：跑了几轮？ |
| **忘记更新变量** | 循环体内变量没变，死循环 | 检查：每次循环是否有变量值被修改？ |
| **初始状态混淆** | 进入循环前和进入后变量值分不清 | 画表格：列名 = 变量，行 = 每轮循环后的值 |

## 一句话总结

> **迭代代码 = 思维上的流水线——你需要在大脑中模拟循环的执行：初值是什么，每轮做了什么，变量怎么变，什么时候停。高考没有编程题，但流程图题、算法描述题的本质就是让你在脑子里跑一遍迭代代码。**
