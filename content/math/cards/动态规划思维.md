# 动态规划思维

> **Intuitive 卡片不教你"怎么算"，而是教你"算什么"。** 动态规划思维在高考中不会直接以"DP"命名登场——它总是穿上"多阶段决策"的包装、藏在递推+最优化的嵌套结构中、或作为新定义的"最优子结构"出现。这张卡片只做一件事：让你在考场 30 秒内认出这是个"不要重复算"的信号，建立状态和转移方程。

---

## 视角层（像什么）

### 核心隐喻：爬山记账——每一步都写在石板上，后面的决策直接查石板

**动态规划 = 边爬山边在石板上刻字——到了新高度，先看石板上"从这往上的最佳路线"有没有人刻过**

```
传统爬山（暴力搜索）：每到一个岔路口，把所有可能的后续路线都走一遍
  → 路线数指数爆炸，一座小山要走到天荒地老

动态规划爬山：每到一个点，算好"从这到山顶的最短路线"，刻在石板上
  → 后面再有别的路经经过这个点，直接查石板，不用重走

       ┌─→ B ─→ D ─→ 山顶
      /
起点 A ─→ C ─→ D ─→ 山顶（重复！D→山顶 在 B 那路已经算过了）
      \
       └─→ E ─→ 山顶

暴力：每条路都走一遍（D→山顶 算了两次）
DP：D→山顶 第一次算完就存起来，第二次直接取值（O(1)）
```

> **一句话直觉：** 动态规划就是"算过的不重算"——把一个"大问题怎么拆分"写成一句话（状态转移方程），然后从最小的子问题开始算，边算边记，后面需要时直接查。

### 动态想象：存钱罐的逐日累积

```
你想知道第 30 天时存了多少钱。

规则：每天存的钱 = 昨天的余额 + 前天余额的一半（某种利息规则）

暴力法：第 30 天 → 需要第 29 和第 28 天 → 每个又需要前两天 → 分支爆炸
DP 法：
  第 1 天：存 10 元 → 记在纸上
  第 2 天：存 10 元 → 记在纸上
  第 3 天：查纸：10 + 10/2 = 15 → 记在纸上
  第 4 天：查纸：15 + 10/2 = 20 → 记在纸上
  ...
  第 30 天：查纸，得结果。总共算了 30 次，不是指数次。
```

**DP 的三要素（存钱罐版）：**

| 要素 | 存钱罐场景 | 数学对应 |
|------|---------|---------|
| **状态定义** | "第 i 天的余额" | dp[i] = 什么？——最关键的一句话 |
| **状态转移方程** | "今天的余额 = 昨天余额 + 前天余额的一半" | dp[i] = f(dp[i-1], dp[i-2], …) |
| **边界条件（初始值）** | 第 1 天和第 2 天没有"前天"，必须单独给 | dp[1] = ?, dp[2] = ? |

### 反直觉警示：大脑会在哪里翻车

| 翻车点 | 直觉告诉你 | 实际真相 |
|--------|-----------|---------|
| **最优子结构** | "每一步选最优，整体就最优" | 只有问题具有"最优子结构"时才对。有些问题（如最长简单路径）每一步贪心选最优会翻车——但 DP 要求子问题最优能推出整体最优，这是它的前提条件 |
| **状态定义瞎写** | "dp[i] 就是第 i 步的结果" | dp[i] 的定义决定了转移方程的形式。定义错一个维度，后面全错。常见的坑：少维度（忘了记"上一状态是什么"）或多维度（增加了无用的冗余信息）|
| **递推方向反** | "从大往小推也一样" | DP 必须从已解决的子问题推向未解决的。如果你定义 dp[i] 依赖于 dp[i+1]，那就得从后往前推。方向选反会导致循环依赖、算不出来 |
| **重叠子问题遗漏** | "这个子问题不会被重复用到吧？" | 看似独立的路径可能在深处汇合。DP 的前提就是子问题会被重复访问——如果每个子问题只用一次，那递推就够了，不需要 DP 的"查表"机制 |

## 联结层（连什么）

### 跨学科锚点

**1. 计算机科学：动态规划是算法设计的核心范式**

```python
# 斐波那契：不用 DP → O(2ⁿ)；用 DP → O(n)
# 不用 DP（暴力递归）
def fib(n):
    if n <= 1: return n
    return fib(n-1) + fib(n-2)  # 每个 fib(k) 被重复计算了指数多次！

# 用 DP（备忘录）
dp = {0:0, 1:1}
def fib(n):
    if n not in dp:
        dp[n] = fib(n-1) + fib(n-2)  # 每个只算一次
    return dp[n]
```

**2. 经济学：动态最优化**

```
家庭如何安排一生的消费和储蓄？

每一年：
  状态：现有资产
  决策：消费多少、存多少
  转移：资产 × (1+利率) - 消费 = 下一年资产

贝尔曼方程（DP 的核心）：
  V(资产) = max{ 效用(消费) + β·V(下一期资产) }

这就是动态规划在经济学中的应用——跨期最优决策。
```

**3. 生物学：序列比对算法**

```
比对两条 DNA 序列 ATCG 和 ATGG：
每一对字符的比对可以拆分为子问题——
1. A 和 A 比对了 → 剩下 TCG 和 TGG 的比对（子问题）
2. A 跳过（插入空位）→ 剩下 TCG 和 ATGG（子问题）
3. G 跳过 → 剩下 ATCG 和 TGG（子问题）

选最优的那个 → 典型的 DP 网格填充问题（Needleman-Wunsch 算法）
```

### 向 Formal 的桥梁

**拆包后，落回哪条 formal 定理？**

| 高考场景 | 拆包结论 | Formal 着陆点 |
|---------|---------|--------------|
| 爬楼梯问题："每次走 1 级或 2 级，到第 n 级有几种走法" | DP：dp[n]=dp[n-1]+dp[n-2] | 斐波那契递推 |
| "将正整数 n 拆分为正整数之和，求拆分方式数" | DP：整数拆分，完全背包变体 | 生成函数 / 递推公式 dp[i][j] |
| "从矩阵左上角到右下角，求最大/最小路径和" | DP 网格：dp[i][j]=a[i][j]+max(dp[i-1][j], dp[i][j-1]) | 二维递推 / 网格 DP |
| "n 个物品，各有价值 v[i] 和重量 w[i]，总重量 ≤ W，求最大价值" | DP：0/1 背包 | 背包问题的递推公式 |
| "新定义的状态转移系统，求最优方案" | 识别 DP 三要素（状态/转移/边界） | 建立 dp 数组并递推 |

**首选工具**：`状态转移方程`——先明确 dp 的定义，再写转移关系，最后推边界。
**次选工具**：`暴力搜索+记忆化`——如果转移方程不好找，先用递归把结构摸清，再加备忘录。

### 概念网络坐标

```
              ┌─────────────────┐
              │    递推/递归    │  ← 通用模式：大问题→小问题
              │ （数学归纳法）  │
              └────────┬────────┘
                       │
                       ▼
              ┌─────────────────┐
              │  动态规划思维   │  ← 优化：小问题有重叠，算过不重算
              │ （状态→转移→边界）│
              └────────┬────────┘
                       │
           ┌───────────┼───────────┐
           │           │           │
           ▼           ▼           ▼
     ┌─────────┐ ┌─────────┐ ┌─────────┐
     │最优路径 │ │ 计数DP │ │可行性DP│
     │求最值   │ │ 计数方案│ │判断可达│
     └─────────┘ └─────────┘ └─────────┘
```

**拆包调用优先级：**
1. 看到"最优/最短/最大/最小" + 多阶段 → 先判断是否有最优子结构，再设 dp
2. 看到"计数/方案数" + 递推结构 → DP 计数，dp[i] = 符合条件的方案总数
3. 看到"新定义转移系统" → 提取三要素：状态是什么、怎么转移、初始条件

### 历史/哲学视角：贝尔曼的"最优性原理"

**动态规划不是"发明"的，是"命名"的——贝尔曼故意起了个模糊的名字。**

```
1950 年代：兰德公司，理查德·贝尔曼
贝尔曼在研究多阶段决策过程时，发现了一个通用原理：
  "最优策略的一部分，本身也是最优策略。"
  （最优性原理）

为什么叫"动态规划"？
贝尔曼的原话："我选择这个名字是为了让国防部长（威尔逊）
无法反对我的研究。'动态'是一个好词，'规划'也是好词。
没有哪个头脑正常的人会对'动态规划'说不。"
↓ 一个为了躲官僚而起的名字，变成了计算机科学最重要的算法范式。
```

> **哲学意义：** DP 的核心思想——"大问题的最优解包含小问题的最优解"——其实是数学归纳法在优化问题中的翻版。你证明 n 最优蕴含 n-1 最优，然后从基值开始步步推导。高考里的 DP 题本质上是"递推 + 择优"——找出每一步的最优选择，并用表格记住中间结果。

## 迁移层（何时调用）

### 三类场景判定表

| 场景 | 识别信号 | 不拆包的代价 | 拆包收益 |
|:---|:---|:---|:---|
| 🎭 **包装** | 应用题："生产计划""最优路径""资源分配""投资策略" | 被商业术语绕晕，试图列方程但变量太多 | 3 秒提取"阶段+状态+决策"，识别为 DP 问题 |
| 🔀 **嵌套** | 两层递推嵌套："选物品放入背包，且物品的可用次数依赖于前一阶段的选择" | 双层嵌套暴力搜索 → 指数复杂度 | 内层 DP（单个阶段的最优选择）→ 外层 DP（阶段间的累积最优） |
| 🆕 **新定义** | "定义一种运算 ⊕，使得 dp[n] = max(dp[n-1]⊕a[n], dp[n-2]⊕b[n])" | 自定义运算让人懵掉 | 验证 ⊕ 的性质 → 如果是加法/乘法/取 min-max，则 DP 套路完全适用 |

### 拆包方向标（10 秒流程）

```
Step 1：识别结构类型（3 秒）
  • 看到"最优 + 多阶段/多步骤/分阶段决策"？
    → 拆"DP 包装"：直接套 DP 三要素模板

  • 看到双层递推（内层和外层各有递推规则）？
    → 拆"嵌套包装"：分内外层，每层独立建 dp 数组

  • 看到"定义一种新的累积规则，求最优…"？
    → 拆"新定义包装"：验证新规则是否满足最优子结构

Step 2：剥外壳找骨骼（4 秒）
  • 定义状态：dp[i] 或 dp[i][j] 代表什么？（最关键的一行文字）
  • 写出转移：dp[当前] = 某种运算(dp[之前状态]) + 当前收益
  • 确定边界：dp[0] 或 dp[1] 等于什么？边界决定了递推的起点

Step 3：锁定 Formal 着陆点（3 秒）
  • 问自己：这是计数 DP、最优 DP、还是可行性 DP？
  • 写下：拆包→【dp 定义 → 转移方程 → 边界 → 递推求解】
```

### 向 Formal 切换铁律

> **拆包完成后，30 秒内必须切换为 formal 语言。Intuitive 只负责定向，不负责计算。**

| 场景 | 拆包结束的标志 | 切换后的 formal 动作 |
|:---|:---|:---|
| 包装题 | dp 定义和转移方程已写出 | 严格递推：for i from 1 to n，按转移方程更新 dp[i] |
| 嵌套题 | 内外层 dp 责任已分离 | 外层递推时调用内层 dp 结果，每次内层重新计算或缓存 |
| 新定义题 | 新运算在 dp 转移中的作用已确认 | 严格写出带自定义运算的转移方程和初始化条件 |

**违反此铁律的后果**：dp 定义模糊，转移方程缺项或多算了不该算的状态，导致递推结果全局错误。

### 直觉陷阱清单（仅拆包场景）

| 陷阱 | 何时失效 | 纠错方式 |
|------|---------|---------|
| **贪心冒充 DP** | 每次选局部最优，跳过 dp 表 | 贪心只在"最优子结构 + 贪心选择性质"同时成立时有效。不确定时——选 DP，因为 DP 更安全 |
| **状态维度不足** | dp[i] 只记了当前一步，忘了前一阶段的选择影响现在 | 如果"当前决策"依赖于"上一步做了什么选择"，dp 至少需要二维：dp[i][j] 表示"第 i 步且上一步选的是 j" |
| **递推方向反** | dp[i] 依赖 dp[i+1]，但递推从 0 开始 | 检查依赖方向：箭头指向谁，就从谁开始推 |
| **忘记初始化负数** | 求最大值，dp 初始化为 0，但答案可以是负的 | 求最大值初始化为 -∞，求最小值初始化为 +∞ |

## 一句话总结

> **动态规划思维是"算过不重算，小步推大步"——把大问题砍成有重叠的小问题，从最小的开始算，边算边记，后面直接查。高考里它藏在"最优路径""资源分配""自定义累积规则"之中。你的任务：30 秒内写出 dp 的定义、转移方程和边界条件，然后跑递推。**
