# 离散迭代算法

> **Intuitive 卡片不教你"怎么算"，而是教你"算什么"。** 有些问题没有漂亮的公式解——但你可以一次次重复简单计算，逐步逼近答案。这就是"迭代"的哲学：不求一步到位，但求步步靠近。

---

## 视角层（像什么）

### 核心隐喻：健身增肌——每次加一点重量，肌肉慢慢变大

**离散迭代算法 = 健身房的渐进超负荷训练——一次举不起 100kg？没关系，今天举 50，明天 52，后天 54…**

```
  第 1 天：举 50kg
  第 2 天：举 52kg（+2，基于昨天的水平）
  第 3 天：举 54kg（+2，基于前天的水平）
  ...
  第 N 天：终于举起 100kg

  迭代公式：w_{n+1} = w_n + Δ（增量）
  每次只在上一步的基础上加一点点
  不是"直接算出 100kg 需要多少天"，而是"一步步走到 100kg"
```

**增肌思维的三条铁律：**

| 铁律 | 健身场景 | 违反的后果 |
|------|---------|-----------|
| **基于上一步**：每次只在上次结果上修改 | 今天的训练重量 = 昨天能举的重量 + Δ | 每次从零开始，永远达不到目标 |
| **反馈驱动**：用上次结果的好坏指导下次 | 上次太轻松？下次多加。上次太累？下次少加 | 盲目迭代，越跑越远 |
| **收敛判断**：什么时候"够好了"就停 | 100kg 举起来了 → 目标达成，停止增重 | 永远不会停，或提前放弃 |

> **一句话直觉：** 离散迭代算法就是"不求一步到位"——每次取上一次的近似结果，用同一个规则算出更好的近似，重复直到足够精确。核心是：迭代公式 + 初始值 + 停止条件。

### 动态想象：牛顿法求 √2

```
  目标：算出 √2 ≈ 1.41421356...

  函数 f(x) = x² - 2，它的正根就是 √2

  牛顿迭代公式：x_{n+1} = x_n - f(x_n)/f'(x_n)
               = x_n - (x_n²-2)/(2x_n)
               = (x_n + 2/x_n) / 2

  从 x₀=1 开始：
  x₁ = (1 + 2/1)/2 = 1.5
  x₂ = (1.5 + 2/1.5)/2 ≈ 1.41667
  x₃ = (1.41667 + 2/1.41667)/2 ≈ 1.41422

  仅仅 3 步就精确到了小数点后 5 位！
  这就是迭代的威力——每一步"自我纠正"，越跑越准
```

### 反直觉警示

| 翻车点 | 直觉告诉你 | 实际真相 |
|--------|-----------|---------|
| **初始值无所谓** | "随便选个初值就行" | 坏的初值可能导致不收敛或收敛到错误的根 |
| **迭代一定能收敛** | "多迭代几次总会准的" | 有些迭代是发散的（越跑越远），有些是振荡的（来回跳） |
| **一步到位更好** | "我就要公式解，不要迭代近似" | 很多问题没有公式解（如五次方程），迭代是唯一出路 |

## 联结层（连什么）

### 跨学科锚点

**1. 计算机图形学：分形（曼德勃罗集）**
```
复数迭代：z_{n+1} = z_n² + c
从 z₀=0 开始，反复迭代
判断序列是收敛还是发散 → 据此给平面上的点染色
→ 生成无比复杂的图案，而算法就是一条简单的迭代公式
```

**2. 物理学：模拟退火**
```
金属冷却时原子排列：
  每次迭代把温度降一点，原子重新排列
  温度高时允许大范围跳动（探索），温度低时只做精细调整（收敛）
  → 最终找到最低能量状态
```

### 向 Formal 的桥梁

| 迭代类型 | 迭代公式 | Formal 着陆点 |
|---------|---------|--------------|
| 不动点迭代 | x_{n+1}=g(x_n) | 牛顿迭代.md, 二分法.md |
| 牛顿法 | x_{n+1}=x_n - f/f' | 牛顿迭代.md, 导数定义.md |
| 数值积分 | S_{n+1}=S_n + f(x_n)·Δx | 微积分.md, 投点法.md |
| 线性递推 | a_{n+1}=p·a_n+q | 递推.md, 数列.md |

### 概念网络坐标

```
  无公式解的问题
      │
      ▼
  迭代公式 + 初始值
      │
      ▼
  重复计算直到满足停止条件
      │
      ├── 收敛 → 得到近似解
      └── 不收敛 → 换初值或换迭代公式
```

## 迁移层（何时调用）

### 三类场景判定表

| 场景 | 识别信号 | 拆包收益 |
|:---|:---|:---|
| 🎭 **包装** | "逐次逼近""反复进行以下操作" | 识别为迭代问题，找迭代公式 |
| 🔀 **嵌套** | 多阶段迭代，不同阶段用不同迭代规则 | 画出迭代阶段图，每段独立 |
| 🆕 **新定义** | "定义序列 x_n 满足 x_{n+1}=f(x_n)" | 这就是迭代定义，试试从初值跑几步 |

### 拆包方向标（10 秒流程）

```
Step 1：识别迭代结构（3 秒）
  • 是否形式为 "新的 = f(旧的)"？
  • 是否重复同样的操作？

Step 2：提取迭代要素（4 秒）
  • 初值 = ?
  • 迭代公式 = ?
  • 停止条件 = ?

Step 3：锁定 Formal 着陆点（3 秒）
  • 不动点迭代 → 牛顿法/二分法
  • 数列递推 → 特征根法/迭代
```

### 直觉陷阱清单

| 陷阱 | 何时失效 | 纠错方式 |
|------|---------|---------|
| **不考虑收敛性** | 拿了公式就迭代，不管会不会发散 | 先检查：|g'(x)|<1 在解附近吗？这是收敛的充分条件 |
| **精度不够就停** | 迭代几次就停，误差还很大 | 设定精度阈值，每次迭代后检查 |x_{n+1}-x_n| |
| **初值全一样** | 不同的根需要不同的初值 | 画函数图像，大致确定根的位置后再选初值 |

## 一句话总结

> **离散迭代算法 = "一步一个脚印走向精确"——你不需要一次算对，只需要每次在上一次的基础上改进，重复直到足够好。牛顿法、二分法、数值积分，都是这个思想的不同形态。核心三要素：初值、迭代公式、停止条件。**
