# 离散时间步长

> **Intuitive 卡片不教你"怎么算"，而是教你"算什么"。** 连续的世界是理想化的——现实中很多东西只能"隔一段时间看一眼"。离散时间步长，就是教你用这种"隔一段时间看一眼"的节奏来分析问题。

---

## 视角层（像什么）

### 核心隐喻：逐帧动画——把连续动作拆成一张张定格画面

**离散时间步长 = 用翻页动画的方式理解变化：不是看电影，而是看幻灯片**

```
  连续变化（电影）              离散变化（幻灯片）
┌────────────────────┐     ┌────┬────┬────┬────┐
│  ╱                 │     │    │    │    │    │
│ ╱  平滑的流动      │     │ t=0│ t=1│ t=2│ t=3│
│╱  无数个中间帧     │     │    │    │    │    │
└────────────────────┘     └────┴────┴────┴────┘
                              每帧之间有一个固定的时间间隔 Δt
```

**逐帧动画思维的三条铁律：**

| 铁律 | 动画场景 | 违反的后果 |
|------|---------|-----------|
| **固定步长**：每次观察之间的时间间隔相同 | 每 0.1 秒拍一张照片 | 步长不一致时公式不统一 |
| **状态快照**：每一帧记录的是那一刻的完整状态 | 第 n 帧记录 t=n·Δt 时的全部信息 | 以为离散信息可以还原连续曲线的所有细节 |
| **递推连接**：上一帧+规则=下一帧 | 第 n 帧 → 动画规则 → 第 n+1 帧 | 写不出递推关系，试图一次性跳到最终状态 |

> **一句话直觉：** 离散时间步长就是"每隔 Δt 拍一张照片"——第 n 张照片的状态，由第 n-1 张照片的状态加上变化规则决定。所有离散分析的核心动作就是：写出递推公式。

### 动态想象：银行复利

```
  你在银行存了 10000 元，年利率 5%，每年计算一次利息：

  时刻 t=0（第 0 帧）：本金 P₀ = 10000
  时刻 t=1（第 1 帧）：P₁ = P₀ × (1+5%) = 10500
  时刻 t=2（第 2 帧）：P₂ = P₁ × (1+5%) = 11025
  ...

  步长 Δt = 1 年
  递推公式：P_{n} = P_{n-1} × 1.05

  连续复利（如果每秒钟都计息）：
  P(t) = 10000 × e^(0.05t)  ← 当 Δt→0 时，离散趋于连续
```

### 反直觉警示

| 翻车点 | 直觉告诉你 | 实际真相 |
|--------|-----------|---------|
| **步长越小越好** | "Δt 取得越小越精确" | Δt 减小意味着步数增加，计算量剧烈上升。需要在精度和效率间平衡 |
| **离散和连续等价** | "离散递推和连续求导差不多" | 离散的差分 Δy/Δx 和连续的导数 dy/dx 在 Δx 很小时才近似相等 |
| **步长可以随意变** | "某一帧用 Δt=1，下一帧用 Δt=2 也行" | 变步长会让递推公式复杂化，除非题目明确要求，否则保持步长一致 |

## 联结层（连什么）

### 跨学科锚点

**1. 计算机图形学：游戏帧率**
```
游戏每秒渲染 60 帧（frame），每帧间隔 Δt = 1/60 秒
第 n 帧的角色位置 = 第 n-1 帧的位置 + 速度 × Δt
这就是"离散时间步长"在游戏引擎中的标准用法
```

**2. 生物学：种群调查**
```
每年统计一次麋鹿数量（Δ=1 年）
N_{n+1} = N_n + 出生数 - 死亡数 + 迁入数 - 迁出数
不能每秒都在数——只能隔年采集离散数据
```

### 向 Formal 的桥梁

| 问题类型 | 拆包方向 | Formal 着陆点 |
|---------|---------|--------------|
| 已知递推公式求通项 | 识别递推类型，找通项公式 | 递推.md, 数列.md |
| 已知变化规律求下一状态 | 用 Δt 离散化写出 a_{n+1}=a_n+Δ×f(a_n) | 差分.md, 递推.md |
| 比较离散与连续 | 令 Δt→0，递推→微分方程 | 微积分.md, 瞬时变化率.md |

### 概念网络坐标

```
  实际问题（如复利、种群、温度变化）
      │
      ▼
  选取合适的时间步长 Δt
      │
      ▼
  写出递推公式：a_{n+1} = f(a_n)
      │
      ├── 求通项公式 → 数列
      ├── 数值模拟   → 迭代计算
      └── Δt→0      → 连续模型（微分方程）
```

## 迁移层（何时调用）

### 三类场景判定表

| 场景 | 识别信号 | 拆包收益 |
|:---|:---|:---|
| 🎭 **包装** | "每年…""每代…""每步…"穿了生活/经济的外衣 | 识别出时间被离散化了，开始找递推规律 |
| 🔀 **嵌套** | 时间步长+状态变化+多参数 | 画出时间线，标出每一帧的状态和参数 |
| 🆕 **新定义** | "定义第 n 步的状态为…" | 这就是在定义递推，立即画时间线 |

### 拆包方向标（10 秒流程）

```
Step 1：确认步长（3 秒）
  • "每年/每次/每代" → Δt=1
  • "每隔 x 分钟" → Δt=x
  • 无明确时间？→ 设 Δt，让每次迭代代表一步

Step 2：写递推关系（4 秒）
  • "第 n+1 次的值 = 第 n 次的值 + 变化量"
  • 变化量 = 变化率 × Δt 或其他规则

Step 3：锁定 Formal 着陆点（3 秒）
  • 能否直接求通项？→ 数列
  • 只能迭代？→ 递推+数值模拟
```

### 直觉陷阱清单

| 陷阱 | 何时失效 | 纠错方式 |
|------|---------|---------|
| **Δ=1 省写 Δt** | 递推公式里默认 Δt=1，但实际不是 | 先明确写出 Δt，代入数值时再化简 |
| **忘记初始状态** | 只写了递推公式，没写 a₀ 的值 | 递推需要起点——a₀ 或 a₁ 必须明确 |
| **步数搞错** | "第 10 年后"是 n=10 还是 n=11？ | 画出时间线，从 n=0 开始逐帧标号 |

## 一句话总结

> **离散时间步长就是把连续时间切成等间隔的"片段"，每个片段拍一张快照。核心动作：确定步长 Δt → 写出递推关系 → 迭代计算或求通项。所有"每年""每代""每步"的问题，本质上都是离散时间步长模型。**
