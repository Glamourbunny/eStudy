# 离散步进

> **Intuitive 卡片不教你"怎么算"，而是教你"算什么"。** "步进"就是一步接一步往前走——不知道终点在哪，但知道每一步怎么走。这就是递推和迭代的本质。

---

## 视角层（像什么）

### 核心隐喻：爬楼梯——每一步都踩在上一步的基础上

**离散步进 = 爬楼梯——你只能站在当前台阶上迈出下一步，不能跳**

```
  第 n 级台阶
      ┌─┐
      │ │  ← 你必须踩过第 n-1 级才能到达这里
      ├─┤
      │ │  ← 第 n-1 级
      ├─┤
      │ │
      ├─┤
      │ │  ← 第 2 级
      ├─┤
      │ │  ← 第 1 级
      └─┘
    起点

  每次只能跨一级（或有限的几级），不能跳也不能飞
  站在第 n-1 级，知道规则，就能到达第 n 级
```

**爬楼梯思维的三条铁律：**

| 铁律 | 爬楼梯场景 | 违反的后果 |
|------|-----------|-----------|
| **步步为营**：只能基于上一步计算下一步 | 必须知道第 n-1 级才能算第 n 级 | 想一次性跳跃到第 n 级，但缺少中间信息 |
| **规则恒定**：每一步的规则是一样的 | 每级台阶高度相同，跨步规则相同 | 每一步用不同的公式，写不出统一的递推式 |
| **方向明确**：往前推（顺推）或往后推（逆推） | 要么从起点往后算，要么从目标往回推 | 顺推逆推混用，自己都搞不清从哪出发 |

> **一句话直觉：** 离散步进就是"一步一个脚印"——第 n 步的状态 = 第 n-1 步的状态 + 步进规则。关键是找到那个"步进规则"（递推公式），然后从起点开始，一步步走到终点。

### 动态想象：斐波那契的兔子

```
  一对兔子从出生后第二个月起，每个月生一对兔子。
  问第 n 个月有多少对兔子？

  第 1 个月：1 对（刚出生，还不能生）
  第 2 个月：1 对（还不能生）
  第 3 个月：2 对（原来的 1 对 + 新生的 1 对）
  第 4 个月：3 对
  第 5 个月：5 对

  步进规则：F_n = F_{n-1} + F_{n-2}
  → 当前 = 上一步 + 上上步
  → 每一步都基于前两步

  你不能直接问"第 100 个月多少对？"然后口算出来
  你必须先算出第 1,2,3,…,99 个月，一步一步来
  这就是"离散步进"
```

### 反直觉警示

| 翻车点 | 直觉告诉你 | 实际真相 |
|--------|-----------|---------|
| **一步登天** | "能不能直接求 F_{100} 而不管前 99 项？" | 有些递推可用特征根求通项（一次性算），有些只能老老实实迭代 |
| **顺序无所谓** | "从第 n 项往前推和从第 1 项往后推是一样的" | 递推有方向！a_n = f(a_{n-1}) 只能顺推；a_{n-1} = g(a_n) 只能逆推 |
| **线性递推全可解** | "所有递推都能用特征根" | 非线性递推（如 a_{n+1}=a_n²+1）没有通项公式，只能迭代 |

## 联结层（连什么）

### 跨学科锚点

**1. 计算机科学：循环迭代**
```python
# 计算斐波那契第 10 项（从第 1 步走到第 10 步）
a, b = 1, 1
for i in range(3, 11):
    a, b = b, a + b  # 每次迭代只基于前两步
```
这就是离散步进的代码实现——循环体内的每一步都只依赖前一步的值。

**2. 日常生活：存钱计划**
```
每个月存 500 元 + 上月余额的 1% 利息
→ 余额_n = 余额_{n-1} × 1.01 + 500
→ 每月一步，一步步累积
```

### 向 Formal 的桥梁

| 步进类型 | 递推形式 | Formal 着陆点 |
|---------|---------|--------------|
| 等差型 | a_n = a_{n-1} + d | 等差数列.md |
| 等比型 | a_n = a_{n-1} × q | 等比数列.md |
| 线性型 | a_n = p·a_{n-1} + q | 递推数列.md |
| 二阶型 | a_n = p·a_{n-1} + q·a_{n-2} | 特征根方程.md |
| 一般型 | a_n = f(a_{n-1}) | 递推.md, 数列.md |

### 概念网络坐标

```
  离散步进（爬楼梯）
      │
      ├── 等差/等比 → 有通项公式，直接算第 n 项
      ├── 线性递推 → 特征根法求通项
      └── 非线性递推 → 只能一步一步迭代
            │
            └── 可编程计算（数值方法）
```

## 迁移层（何时调用）

### 三类场景判定表

| 场景 | 识别信号 | 拆包收益 |
|:---|:---|:---|
| 🎭 **包装** | "第 n 次操作后""经过 n 轮""第 n 代" | 识别为递推问题，画楼梯图找步进规则 |
| 🔀 **嵌套** | 多阶段递推，每个阶段有不同规则 | 拆成多个单独的递推，每段独立求解 |
| 🆕 **新定义** | "定义序列 a_n 满足 a_{n+1}=…" | 这就是在定义步进规则，先试试写出前几项找规律 |

### 拆包方向标（10 秒流程）

```
Step 1：确认步进方向（3 秒）
  • 已知 a_1？→ 顺推（从 a_1 往后算）
  • 已知 a_n？→ 逆推（从 a_n 往前算）

Step 2：写出递推公式（4 秒）
  • a_n = f(a_{n-1}) 或 a_{n-1} = g(a_n)
  • 算前 3-5 项验证公式

Step 3：锁定 Formal 着陆点（3 秒）
  • 等差/等比 → 直接套通项公式
  • 线性 → 特征根法
  • 其他 → 迭代或编程
```

### 直觉陷阱清单

| 陷阱 | 何时失效 | 纠错方式 |
|------|---------|---------|
| **下标混淆** | a_{n+1}=f(a_n) 写成 a_n=f(a_{n+1}) | 始终检查"等号左边是下一步，右边是当前步" |
| **第一项用错** | 等比数列 a_n=a_1·q^{n-1} 写成了 a_n=a_1·q^n | 代入 n=1 检验：a_1=a_1·q^0=a_1 ✓ |
| **忽视边界** | 递推式里分母有 a_{n-1}，但 a_{n-1}=0 时失效 | 检查递推定义域：能不能出现使分母为零的项？ |

---

## 一句话总结

> **离散步进就是"每一步踩在前一步上"——核心是找递推关系 a_n = f(a_{n-1})，然后从起点出发一步步走到目标。能求通项就一次性算，不能就老老实实迭代——两种方法都是"步进"，只是效率不同。**
