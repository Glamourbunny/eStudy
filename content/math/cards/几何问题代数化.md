# 几何问题代数化

> **Intuitive 卡片不教你"怎么算"，而是教你"算什么"。** 几何图形看着很直观，但一旦涉及精确计算，图形的直观就变成了模糊的"差不多"——这时候你需要把图形翻译成代数，用数字的精准裁决几何的是非。

---

## 视角层（像什么）

### 核心隐喻：精密测量仪——用数字"称量"图形的每一寸

**几何问题代数化 = 给几何图形装上一套精密测量系统**

```
  几何直觉（模糊）              代数表达（精确）
┌────────────────────┐     ┌────────────────────┐
│  这个三角形像等腰   │     │  AB = AC = 5.000   │
│  这条线像垂直      │     │  k₁·k₂ = -1       │
│  这个点在圆内      │     │  (x-a)²+(y-b)²<r² │
│  两圆相切...大概？ │     │  联立方程 Δ = 0    │
└────────────────────┘     └────────────────────┘

  "看着像"不可靠              "等式说"才是判决
```

**精密测量仪的三条铁律：**

| 铁律 | 场景 | 违反的后果 |
|------|------|-----------|
| **建系是起点**：没有坐标系，几何只是"看图说话" | 选原点、选方向、定单位——前 10 秒决定后续 10 分钟 | 坐标系选得烂，方程复杂到算不出来 |
| **坐标是翻译**：每个点、每条线都要有代数身份 | 设 A(x₁,y₁)，l: y=kx+b | 遗漏某个点的坐标变量，后续无法计算 |
| **等式是法官**：用等式判断所有几何关系 | 垂直→斜率积为-1，相切→Δ=0，共线→向量成比例 | 靠眼睛判断几何关系，一错全错 |

> **一句话直觉：** 几何说出"长得像"的话，代数给出"就是这样的"判决——建立坐标系，把所有"像"变成"等于"或"不等于"。

### 动态想象：坐标系的"选址"

建系就像开店选址——选得好，后面的计算行云流水；选得差，一路磕磕绊绊。

```
  题目给你这个图形：           你的坐标系选择：

  等腰三角形 ABC              方案A（好）：底边 BC 在 x 轴上，
  AB = AC                     顶点 A 在 y 轴上（对称建系）
                               → 坐标：B(-a,0), C(a,0), A(0,h)
                               → 计算量：极少

                              方案B（差）：A 在 (0,0)，B 在 (3,0)
                               → 坐标：A(0,0), B(3,0), C(x,y)
                               → 还要附加 AB=AC 的条件
                               → 计算量：翻倍

  两条垂直的直线 l₁ ⊥ l₂      方案A（好）：l₁ 为 x 轴，l₂ 为 y 轴
                              方案B（差）：两条都是斜线
```

**坐标系选址口诀**：对称轴放坐标轴上，直角放坐标轴上，关键点放原点上。

### 反直觉警示

| 翻车点 | 直觉告诉你 | 实际真相 |
|--------|-----------|---------|
| **坐标系唯一** | "随便建个系就行" | 坐标系的选择极大影响计算复杂度——好坐标节约 50% 时间 |
| **几何条件可忽略** | "画完坐标系，原来的几何条件就不用了" | 几何条件是方程的来源。建系后要逐条翻译成等式 |
| **消元就完事** | "把方程联立消元就行" | 消元前先化简：利用对称性减少变量数，利用特殊位置简化系数 |

## 联结层（连什么）

### 跨学科锚点

**1. GPS 定位：地球上的几何→卫星的代数**
```
地球表面的位置（几何）──经纬度──→ (φ, λ) 两个数字（代数）
两个城市之间的距离（几何）──球面距离公式──→ 用坐标算出的数值
```
没有"几何代数化"，就没有 GPS 导航。

**2. 建筑工程：设计图→施工数据**
```
设计图上画一条弧线（几何）
→ 翻译成：圆心(15.00, 20.00)，半径 8.50m（代数）
→ 工人按数字施工
```

### 向 Formal 的桥梁

| 几何条件 | 代数翻译 | Formal 着陆点 |
|---------|---------|--------------|
| 两直线垂直 | k₁·k₂ = -1 | 解析几何.md, 直线.md |
| 点在线段中点 | (x₁+x₂)/2, (y₁+y₂)/2 | 坐标运算.md |
| 三点共线 | 向量成比例 / 斜率相等 | 平面向量.md |
| 点 P 在以 AB 为直径的圆上 | PA·PB = 0（向量内积） | 数量积.md |
| 两圆相切 | 圆心距 = |r₁±r₂|，联立 Δ=0 | 圆.md |

```
  几何图形

  建立坐标系（选址策略）
      │
      ▼
  几何条件 ──翻译──→ 代数方程

  向量方法          联立求解
      │                  │
      └────────┬─────────┘

```

## 迁移层（何时调用）

### 三类场景判定表

| 场景 | 识别信号 | 拆包收益 |
|:---|:---|:---|
| 🎭 **包装** | 几何题穿了"物理/工程"的外衣 | 剥掉情境，只剩下点线面，建系列方程 |
| 🔀 **嵌套** | 动点+几何约束+参数层层嵌套 | 先设动点坐标，把每个几何约束翻译成等式，联立求解 |
| 🆕 **新定义** | "定义轨迹 Γ 为满足…的点 P 的集合" | 设 P(x,y)，把定义条件直接翻译成 x,y 的方程 |

### 拆包方向标（10 秒流程）

```
Step 1：扫描图形（3 秒）
  • 有对称轴？→ 对称轴放坐标轴上
  • 有直角？→ 直角顶点放原点，两边放坐标轴上
  • 有关键点？→ 尽量让它坐标为 (0,0) 或 (a,0)

Step 2：逐条翻译（4 秒）
  • 垂直 → 斜率积为 -1
  • 共线 → 向量成比例
  • 距离相等 → 两点距离公式相等
  • 相切 → 联立方程 Δ=0

Step 3：锁定 Formal 着陆点（3 秒）
  • 问自己：这是解析几何的哪种题型？
  • 写下联立方程，开始严格代数推导
```

### 直觉陷阱清单

| 陷阱 | 何时失效 | 纠错方式 |
|------|---------|---------|
| **建系偷懒** | 随便选坐标系导致方程复杂 | 第一个 30 秒只做选址——好的坐标系让方程天然简化 |
| **漏翻译** | 忘记翻译某个几何条件 | 列出所有几何条件，逐条打勾确认已翻译 |
| **算完不翻译回来** | 代数算出 x=3 但题目问"P 点在何处" | 把代数结果翻译回几何语言：P 点坐标为 (3, 某值) |

## 一句话总结

> **几何问题代数化 = 把"看着像"升级为"算出来就是"——建坐标系是第一步也是最重要的一步，好坐标系让方程自己变简单。几何说出"像"，代数拍板"是"。**
