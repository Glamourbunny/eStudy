# 乘法原理

> 乘法原理是"分步相乘"——完成一件事要 n 步，每步分别有 a₁, a₂, ..., aₙ 种方法，则总共有 a₁ × a₂ × ... × aₙ 种。

---

## 概念层

### 问题的起源

穿衣服有讲究：先选上衣（3种），再选裤子（4种）。

问：有多少种穿法？

这是**分步**问题——只有两步都完成了，整个事情才算完成。

第一步 3 种 × 第二步 4 种 = 12 种穿法。

这就是**乘法原理**。

> **一句话：乘法原理是"分步计数"——各步方法数相乘。**

---

### 原始定义（教材原文）

完成一件事要分成 n 个步骤，若第 1 步有 a₁ 种方法，第 2 步有 a₂ 种方法，...，第 n 步有 aₙ 种方法，且**每一步都缺一不可**，则完成这件事共有 **a₁ × a₂ × ... × aₙ** 种不同的方法。

> 出处：人教版《普通高中教科书·数学（A版）选择性必修三》第 1-2 页

---

### 本质定义

**乘法原理的本质是"分步"：**

```
完成一件事
    │
    ├── 第1步（a₁种）──┐
    ├── 第2步（a₂种）──┤ 必须全部完成
    └── 第n步（aₙ种）──┘

总方法数 = a₁ × a₂ × ... × aₙ
```

**关键特征：**
- 各步**相互依存**（缺一步则整个事情未完成）
- 每步是完成事情的**必要条件**

---

### 定义的三要素

| 要素 | 说明 |
|------|------|
| **分步** | n 个步骤，缺一不可 |
| **每步方法数** | a₁, a₂, ..., aₙ |
| **乘法** | 总数 = 各步相乘 |

---

### 与加法原理的对比

| 对比项 | 加法原理 | 乘法原理 |
|--------|---------|---------|
| 核心 | 分类 | 分步 |
| 关系 | 各类互斥 | 各步依存 |
| 公式 | 总数 = Σaᵢ | 总数 = Πaᵢ |
| 例子 | 去上海：飞机或火车或汽车 | 穿衣服：先上衣再裤子 |

## 命题层

### 核心定理

**定理：乘法原理**

```
完成一件事要分 n 步
第 i 步有 aᵢ 种方法（i = 1, 2, ..., n）
且每步缺一不可
则总方法数 = Πaᵢ = a₁ × a₂ × ... × aₙ
```

---

### 典型例题

**例题：分步计数**

```
题目：从 1, 2, 3, 4, 5 中取两个不同的数组成两位数，有多少种？
```

**深层分析：**

```
分两步：
第1步：选十位数字（5种，1~5都可）
第2步：选个位数字（4种，不能和十位相同）

总方法数 = 5 × 4 = 20 种
```

> **方法提炼：** 分步相乘。

## 方法层

### 识别信号表

| 关键词 | 信号 | 方法 |
|--------|------|------|
| "然后""再" | 分步 | 用乘法原理 |
| "第1步""第2步" | 分步 | 各步相乘 |
| "缺一不可" | 分步特征 | 每步都是必要的 |

---

### 方法详解：加法原理 vs 乘法原理

**Step 1: 识别关键词**

```
"或" → 加法
"然后""再" → 乘法
```

**Step 2: 判断是分类还是分步**

```
分类：各类互斥，任选一类就能完成
分步：各步依存，必须全部完成
```

**Step 3: 执行加法或乘法**

## 常见误区与纠正

| 编号 | 常见误区 | 错误根源 | 正确理解 |
|------|---------|---------|---------|
| 1 | "加法和乘法混淆" | 混淆分类分步 | 互斥用加法，依存用乘法 |
| 2 | "分步不完整" | 粗心 | 确保每步都考虑到了 |
| 3 | "顺序影响结果但用加法" | 误解 | 顺序要用乘法，如排列 |
