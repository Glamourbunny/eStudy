# 逼近

> **Intuitive 卡片不教你"怎么算"，而是教你"算什么"。** 逼近是连接"算不出"和"可以算"的桥——当精确值遥不可及时，逼近会在有限步内给出任意精度的答案。高考中它藏在牛顿迭代、二分法、泰勒近似的背后，常被包装成"估算""近似"或新定义的迭代算法。这张卡片让你认清逼近的本质逻辑。

---

## 视角层（像什么）

### 核心隐喻：雕塑家的减法

**逼近 = 雕塑家从原石开始，一刀一刀削减误差，直到形状"够准"为止**

```
  原石（初始猜测）→ 第 1 刀 → 第 2 刀 → 第 3 刀 → … → 雕像（目标值）

  每走一步，误差缩小，但这个缩小有三种模式：

  二分逼近（恒速收敛）：
     误差/2 → 误差/4 → 误差/8 → ...
     每刀削掉一半——慢但稳，六刀砍到 1/64
  
  牛顿逼近（加速收敛）：
     误差 → 误差² → 误差⁴ → 误差⁸ → ...
     每刀削掉的量跟"剩余误差的平方"成正比——超快，但刀法不稳定
  
  泰勒逼近（局部贴合）：
     常数贴合 → 切线贴合 → 二次曲线贴合 → ...
     从"只能在一小段贴合"逐步扩大到更大范围
```

**逼近的三条直觉铁律：**

| 铁律 | 雕塑类比 | 数学含义 |
|------|---------|---------|
| **误差递减** | 每次下刀后，剩的材料只少不多 | 迭代序列的残差单调减小（或稳定收敛） |
| **速度分等级** | 粗切削快于精抛光——但不同刀法的切削速度天差地别 | 线性收敛 vs 二次收敛 vs 超线性收敛 |
| **够用就停** | 雕像不需要精确到分子级别——眼睛看不出来就够了 | |x_n - x*| < ε 时停止迭代 |

> **一句话直觉：** 逼近就是"猜一个初始值，然后根据误差的方向和大小，一步步修正到够用为止"——雕塑家在黑暗中摸索，每次下刀前先摸一下现状。

### 动态想象：黑暗中找电灯开关

把逼近想象成在完全黑暗的房间里找墙上的电灯开关：

```
  策略 1（二分法）：
    先摸房间的左端和右端，确认开关在中间某处
    然后走到中点摸一下——开关在左边还是右边？
    淘汰一半区域，重复。
    每次走的路都一样远，稳定但不够聪明。

  策略 2（牛顿法）：
    在当前位置摸墙的曲率（切线方向）
    沿着切线方向直走——假设墙是平的
    到达切线与墙的交点附近——比上一步近得多！
    但如果墙的形状太弯，切线会把你带偏。

  策略 3（割线法）：
    取最近两次触摸的位置，连线并延伸
    用这两个点"猜"出方向
    不需要摸曲率（避免牛顿法的代价）
```

**黑暗中找开关在高考里的翻版：**
- "用二分法求方程 f(x)=0 的近似解"——二分法在区间 [a,b] 内反复折半
- "用牛顿法求 √2 的近似值"——从 x₀=1 开始迭代 x_{n+1} = (x_n + 2/x_n)/2
- "证明迭代序列收敛"——证明误差递减 + 有界 / 柯西收敛准则

### 反直觉警示：大脑会在哪里翻车

| 翻车点 | 直觉告诉你 | 实际真相 |
|--------|-----------|---------|
| **初值决定命运** | "牛顿法这么厉害，随便猜一个就能收敛吧？" | **初值太远或靠近拐点，牛顿法会发散甚至振荡，永远不收敛！** |
| **快不等于好** | "收敛越快的方法越应该优先用" | **牛顿法每次迭代要算导数，计算代价可能远超二分法。快速收敛可能被高步均摊成本抵消** |
| **近似不是错误** | "近似解不算解，只是差点意思" | **绝大多数真实问题没有精确解——工程里逼近 = 解。天宫空间站靠迭代算出来的轨道飞得好好的** |
| **区间包含性** | "二分法的前提 f(a)f(b)<0，只是个小条件" | **这个条件就是零点定理本身漏了它则二分法失效——可能区间内有偶数个零点而 f(a)f(b)>0** |

## 联结层（连什么）

### 跨学科锚点

**1. 物理：理论物理中的微扰论**

```
氢原子能级：精确解可用薛定谔方程严格求出
氦原子能级：无法精确求解！

物理学家用微扰论：
  E ≈ E₀ + E₁·ε + E₂·ε² + E₃·ε³ + ...

每一项都是对前一项的"逼近修正"——这就是泰勒逼近在量子力学里的应用。
取到一阶微扰给粗糙答案，取到二阶更精确——取多少阶，取决于你需要的精度。
```

**2. 计算机：机器学习里的梯度下降**

```
梯度下降 = 多维版本的牛顿法简化版

目标：找到函数的最小值
每次迭代：沿着梯度的反方向走一小步（学习率 α）
  x_{n+1} = x_n - α·f'(x_n)

这就是逼近思想的计算机实现——看不到全局，只靠"脚下的坡度"一步步走向谷底。
```

**3. 生活：GPS 定位修正**

```
GPS 初始定位：可能有 10 米的误差
第 1 次修正（用 4 颗卫星）：误差 → 3 米
第 2 次修正（用更多卫星）：误差 → 1 米
第 3 次修正（差分 GPS）：误差 → 0.1 米

每一步都在逼近真实位置——逼近无处不在。
```

### 向 Formal 的桥梁

**拆包后，落回哪条 formal 定理？**

| 高考场景 | 拆包结论 | Formal 着陆点 |
|---------|---------|--------------|
| 二分法求方程近似解 | 二分迭代：每次区间减半 | 牛顿迭代.md（二分法） |
| 牛顿法求近似解 | 切线逼近：x_{n+1} = x_n - f/f' | 牛顿迭代.md |
| 用切线近似函数值 | 线性近似：f(x)≈f(a)+f'(a)(x-a) | 泰勒展开.md + 导数定义.md |
| 证明迭代数列收敛 | 单调有界 / 柯西收敛 | 数列.md + 递推.md |
| 新定义"逼近算法" | 三要素：初始值/迭代规则/停止条件 | 牛顿迭代.md + 二分法.md |

**首选工具**：`误差递减法`——不论什么逼近算法，核心一定是证明误差在缩小（或至少不增大并趋于零）。

**次选工具**：不动点迭代框架——把 x_{n+1}=g(x_n) 视为不动点问题，用压缩映射原理判断收敛。

**拆包调用优先级：**
1. 见到"求近似解""估算""近似值" → 逼近场景

### 历史/哲学视角：逼近是"接受不完美"的科学革命

牛顿发明微积分时，用的是"无穷小量"——一个既是 0 又不是 0 的概念。贝克莱主教嘲讽它是"消逝量的鬼魂"。直到柯西把微积分的根基建立在极限（ε-δ）之上，人们才接受：**完美精确是不可达的，但我们可以无限逼近它。**

这一思想革命影响了整个人类文明——天气预报用迭代求解纳维-斯托克斯方程，航天器用数值方法计算轨道，AI 用梯度下降训练模型。人类接受了"逼近即真理"，才打开了现代科学的闸门。

## 迁移层（何时调用）

### 三类场景判定表

| 场景 | 识别信号 | 不拆包的代价 | 拆包收益 |
|:---|:---|:---|:---|
| 🎭 **包装** | "试估算""近似计算""精确到小数点后…"；迭代公式的物理背景；测量修正式 | 不知道用哪种逼近策略 | 识别精度要求，选二分/牛顿/泰勒 |
| 🔀 **嵌套** | 外层逼近嵌套内层迭代；x_{n+1} = g(x_n, f(x_n)) 两层结构 | 每层收敛条件纠缠，分析混乱 | 分离层：外层控制精度，内层控制单步计算 |
| 🆕 **新定义** | "定义迭代算子 T…""若序列 x_n 由…生成"——一种新的逼近方式 | 不理解收敛条件，无法分析 | 套用逼近三要素（初值/规则/停止条件）快速分析 |

### 拆包方向标（10 秒流程）

```
Step 1：识别结构类型（3 秒）
  • 看到 f(a)f(b)<0 + 求根 → 二分法场景！
  • 看到 x_{n+1} = x_n - f/f' → 牛顿法场景！
  • 看到"在 x=a 附近展开" → 泰勒逼近场景！

Step 2：剥外壳找骨骼（4 秒）
  • 精度要求（小数点后几位）→ 决定迭代次数
  • 初值试探——是不是在收敛域内？
  • 嵌套迭代：外层和内层分别的收敛条件

Step 3：锁定 Formal 着陆点（3 秒）
  • 问自己：什么类型的逼近？收敛速度如何？
  • 写下：拆包→【二分法/牛顿法/泰勒展开/不动点迭代】，立即切换严格语言
```

### 向 Formal 切换铁律

> **拆包完成后，30 秒内必须切换为 formal 语言。Intuitive 只负责定向，不负责计算。**

| 场景 | 拆包结束的标志 | 切换后的 formal 动作 |
|:---|:---|:---|
| 包装题 | 逼近策略已选定 | 按算法步进：f(mid)=..., 判断符号更换区间 |
| 嵌套题 | 层层逼近的依赖关系已分离 | 每层独立做迭代/泰勒展开 |
| 新定义题 | 新算法的三要素（初值/规则/停止）已确认 | 按定义步进，验证收敛条件 |

**违反此铁律的后果**：迭代方向搞反或在迭代公式中符号错误，导致逼近发散而非收敛。

### 直觉陷阱清单（仅拆包场景）

| 陷阱 | 何时失效 | 纠错方式 |
|------|---------|---------|
| **牛顿法发散** | 初值在拐点附近或距根太远 | 先用二分法缩小范围，再用牛顿法加速 |
| **二分法无限** | f(a)f(b)<0 但区间内有偶数个根 | 分区间检查，零点定理只保证至少一个根 |
| **精度≠小数位数** | 要求"0.001 精度"和"保留 3 位小数"不是同一回事 | 精度看绝对误差；小数位数看输出格式 |
| **隐式迭代混淆** | 方程嵌套了自身，写成 x = f(x) 才看出不动点 | 把方程化为 x = g(x) 形式，分析 g 的性质 |

## 一句话总结

> **逼近是"雕塑家的减法"——猜一个初值，根据误差调整，一步步磨掉差距。高考里它藏在牛顿迭代里（乘法变加法）、二分法里（折半折半再折半）、泰勒展开里（用多项式贴身模仿）。你需要 30 秒识别这是哪种逼近范式，看清收敛条件，然后交给 Formal 一步步迭代计算。**
