# 递推

> 递推是"用前面的结果推出后面的结果"——给定初始值和递推关系，就能一步步求出所有结果。

---

## 概念层

### 问题的起源

有些问题难以直接给出通项公式，但容易知道相邻项之间的关系。

例如：斐波那契数列 1, 1, 2, 3, 5, 8,... 每一项都是前两项之和。我们不知道第100项是多少，但通过递推关系一步步算下去总能求出来。

这就是**递推**的思想——用"已知"推"未知"。

> **一句话：递推是根据已知项找出未知项的思维方式。**

---

### 本质定义

**递推的核心结构：**

```
初始项：a₁ = 已知值
递推关系：aₙ₊₁ = f(aₙ) 或 aₙ₊₂ = g(aₙ, aₙ₊₁)
```

**两种基本类型：**

| 类型 | 递推关系 | 例子 |
|------|---------|------|
| 一阶递推 | aₙ₊₁ = f(aₙ) | aₙ₊₁ = 2aₙ |
| 二阶递推 | aₙ₊₂ = f(aₙ, aₙ₊₁) | aₙ₊₂ = aₙ₊₁ + aₙ |

---

### 递推与数列的关系

| 表述 | 说明 |
|------|------|
| **递推公式** | 描述相邻项关系 |
| **通项公式** | 直接给出 aₙ 的表达式 |
| **递推→通项** | 从递推求通项是重要技能 |

## 命题层

### 核心定理

**定理：一阶线性递推**

```
aₙ₊₁ = r·aₙ
则 aₙ = a₁·rⁿ⁻¹
```

**定理：斐波那契数列的通项**

```
aₙ₊₂ = aₙ₊₁ + aₙ
通项：aₙ = (φⁿ - (-φ)⁻ⁿ)/√5
其中 φ = (1+√5)/2 为黄金分割率
```

**定理：递推与特征根方程**

对于二阶线性递推 aₙ₊₂ = p·aₙ₊₁ + q·aₙ：
- 特征方程：r² = pr + q
- 若 r 是特征根，则 aₙ = C·rⁿ

---

### 递推的收敛性

**定理：收敛递推**

```
若 0 < |r| < 1，则 aₙ₊₁ = r·aₙ 收敛到 0
若 r = 1，则 aₙ₊₁ = aₙ 收敛到 a₁
若 |r| > 1，则发散
```

**应用：** 判断数列是否趋向稳定值。

### 方法详解：解递推问题的步骤

**Step 1: 识别递推类型**

```
看递推关系涉及几项
一阶：aₙ₊₁ = f(aₙ)
二阶：aₙ₊₂ = f(aₙ, aₙ₊₁)
```

**Step 2: 确定初始值**

```
给出 a₁ 或 a₀，确保递推有起点
```

**Step 3: 求通项或迭代**

```
一阶线性：用指数公式
二阶线性：用特征根方程
非线性：尝试变形或归纳
```

**Step 4: 检验**

```
代入前几项验证
```

### 常见递推模型的解法

**等比递推：**

```
aₙ₊₁ = r·aₙ
解：aₙ = a₁·rⁿ⁻¹
```

**等差递推：**

```
aₙ₊₁ = aₙ + d
解：aₙ = a₁ + (n-1)d
```

**线性齐次递推：**

```
aₙ₊₂ = p·aₙ₊₁ + q·aₙ
解法：特征方程 r² = pr + q
- 两个不同根 r₁, r₂ → aₙ = A·r₁ⁿ + B·r₂ⁿ
- 两个相同根 r → aₙ = (A+Bn)·rⁿ
```

**线性非齐次递推：**

```
aₙ₊₁ = r·aₙ + c
解：aₙ = (a₁ - c/(1-r))·rⁿ⁻¹ + c/(1-r)
```

## 常见误区与纠正

| 编号 | 常见误区 | 错误根源 | 正确理解 |
|------|---------|---------|---------|
| 1 | "忽略初始值" | 粗心 | 递推必须有起点 |
| 2 | "递推项数混淆" | 粗心 | aₙ₊₁ 意味着往后推一步 |
| 3 | "所有递推都能求通项" | 误解 | 非线性递推可能无法求通项 |
| 4 | "忘记检验收敛性" | 误解 | 发散递推没有极限 |
