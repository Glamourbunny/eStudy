# 归纳猜想

> **Intuitive 卡片不教你"怎么算"，而是教你"算什么"。** 题目给了前几项或者前几个数据点，让你猜出后面的规律——这不是"瞎猜"，而是一种有章法的思维模式。

---

## 视角层（像什么）

### 核心隐喻：拼图游戏——从零散的碎片中拼出完整的图案

**归纳猜想 = 玩拼图——给你几片碎片，你要推断出整张图的模样**

```
  给你的碎片                    你要拼出的完整规律
┌─────┬─────┬─────┐        ┌─────────────────────┐
│ f(1)│ f(2)│ f(3)│        │   猜出 f(n) 的公式  │
│ = 1 │ = 4 │ = 9 │  ───→ │   f(n) = n² ？      │
└─────┴─────┴─────┘        │   验证 n=4: f(4)=16 │
                           │   确实符合！         │
                           └─────────────────────┘

  碎片是"数据"                规律是"通项公式"
```

**拼图思维的三条铁律：**

| 铁律 | 拼图场景 | 违反的后果 |
|------|---------|-----------|
| **找差找比**：先看相邻项的差或比是否有规律 | 碎片之间有什么关系？是不是缺了同一块形状的？ | 盯着原始数据干瞪眼 |
| **前验证后推广**：猜出规律后必须用新的数据点验证 | 猜完整图后，拿手里还多的一片对对看 | 猜了个错的规律就继续用 |
| **猜想不是证明**：归纳出来的结果需要数学归纳法严格证明 | 拼图猜对了≠拼图的每一片都是用正确方法放上去的 | 以为猜对就等于证完了 |

> **一句话直觉：** 归纳猜想不是瞎蒙——给了一堆数据，先算相邻项的"差"和"比"，找差是常数（等差）、比是常数（等比）、差的变化有规律（二阶等差）。猜出规律后，用给的下一项验证，猜对了再用数学归纳法证明。

### 动态想象：从数据中发现平方数规律

```
  题目：a₁=1, a₂=4, a₃=9, a₄=16, 求通项公式

  方法1：直接看（有经验的人）
  1, 4, 9, 16 → 1², 2², 3², 4² → a_n = n²

  方法2：差的方法（没经验的人也能用）
  数列：1,  4,  9,  16
  一阶差： 3,  5,  7    ← 不是常数
  二阶差：   2,   2      ← 常数！二阶等差数列
  → a_n = An² + Bn + C，代入前三项求 A,B,C
  → A=1, B=0, C=0 → a_n = n²

  一阶差和二阶差是发现规律的"通用算法"
  不知道规律长什么样？没关系——差的方法会告诉你
```

### 反直觉警示

| 翻车点 | 直觉告诉你 | 实际真相 |
|--------|-----------|---------|
| **只看前几项就能确定规律** | "1,2,3 后面肯定是 4" | 可能是 1,2,3,5（素数+1），1,2,3,6（阶乘）——同一个开头可能有无数种延续 |
| **多项式一定对** | "前 N 个点总能插值出一个 N-1 次多项式" | 这个多项式可能完全偏离真实规律——猜想需要通过多个数据点验证 |
| **等差/等比就够** | "数列规律不是等差就是等比" | 斐波那契、平方数、立方数……大量常见数列都不是等差或等比 |

## 联结层（连什么）

### 跨学科锚点

**1. 物理学：开普勒第三定律的发现**
```
第谷·布拉赫观测了数十年的行星位置数据（碎片）
开普勒分析这些数据（拼图）：
  水星：T=0.24年, a=0.39天文单位
  金星：T=0.62年, a=0.72天文单位
  地球：T=1.00年, a=1.00天文单位
  火星：T=1.88年, a=1.52天文单位

  计算 T²/a³：
  0.24²/0.39³≈1, 0.62²/0.72³≈1, 1²/1³=1, 1.88²/1.52³≈1
  → 猜想：T²/a³ = 常数！
  → 这就是开普勒第三定律的发现过程——纯粹从数据中归纳出来的
```

**2. 生物学：DNA 双螺旋的发现**
```
罗莎琳·富兰克林的 X 射线衍射照片是"数据"
沃森和克里克从中归纳出"双螺旋结构"是"猜想"
后来的实验验证了这个猜想
```

### 向 Formal 的桥梁

| 归纳场景 | 策略 | Formal 着陆点 |
|---------|------|--------------|
| 等差/等比数列 | 一阶差为常数 / 一阶比为常数 | 等差数列.md, 等比数列.md |
| 二阶等差数列 | 二阶差为常数 → a_n = An²+Bn+C | 数列.md, 求和.md |
| 斐波那契型 | a_n = p·a_{n-1} + q·a_{n-2} | 递推.md, 特征根方程.md |
| 递推找规律 | 算前 5-6 项，找差/比/结构 | 递推.md |

### 概念网络坐标

```
  数据点（前几项/前几个值）
      │
      ▼
  计算一阶差/一阶比
      │
      ├── 常数 → 等差/等比数列
      ├── 再算二阶差
      │     ├── 常数 → 二次多项式
      │     └── 再算三阶差 → 三次多项式…
      └── 无规律 → 试试递推关系 a_n = f(a_{n-1})
```

## 迁移层（何时调用）

### 三类场景判定表

| 场景 | 识别信号 | 拆包收益 |
|:---|:---|:---|
| 🎭 **包装** | "观察前几组数据，推测规律"穿了实验的外衣 | 去掉实验背景，只剩数列，用差/比法找规律 |
| 🔀 **嵌套** | 多组数据，每组各有其规律 | 逐组独立分析，再找组间关系 |
| 🆕 **新定义** | "定义序列…计算其前几项，猜想通项" | 先算出前 4-6 项（代具体值），再用归纳法找规律 |

### 拆包方向标（10 秒流程）

```
Step 1：列出已知项（3 秒）
  • 把所有已给的数据排成一行
  • 给每一项标注序号 n=1,2,3…

Step 2：计算差或比（4 秒）
  • 相邻项差：Δ_n = a_{n+1} - a_n
  • 相邻项比：r_n = a_{n+1} / a_n
  • 分析差/比的规律

Step 3：猜想+验证（3 秒）
  • 提出猜想公式
  • 用给的下一项或代入计算下一项验证
  • 对上了 → 猜想成立，用数学归纳法证
  • 没对上 → 回去改猜想
```

---

### 直觉陷阱清单

| 陷阱 | 何时失效 | 纠错方式 |
|------|---------|---------|
| **只给 3 项就下结论** | 三个点可以插值出无数种公式 | 至少需要 4-5 项才能比较可靠地猜想 |
| **差的方法用过头** | 差不是常数就放弃 | 再算二阶差、三阶差——可能是多项式 |
| **忘记验证** | 猜了公式就用，不验证 | 必须用下一个数据点验证猜想 |

---

## 一句话总结

> **归纳猜想 = 给数据找规律——先算相邻项的差（等差？差变成等差？），再算相邻项的比（等比？），都不行就找递推关系。猜出来必须用新数据验证，验证通过再用数学归纳法证明。差和比是两把通用钥匙。**
