# 线性拟合

> **Intuitive 卡片不教你"怎么算"，而是教你"算什么"。** 线性拟合在高考中频繁出现——给定散落的成对数据点，你要找一条"最佳直线"穿过它们。它的本质不是"画线"，而是"找最不冤枉的那根直线"。

---

## 视角层（像什么）

### 核心隐喻：拉直的橡皮筋——找最不冤枉的那根线

**线性拟合 = 在一堆散点中拉一根橡皮筋，让它松手后刚好停在"两边绷力抵消"的位置**

```
  散点分布：                  最佳拟合线：
  
  y ↑        ●                 y ↑      ╱
    │     ●    ●                 │    ╱  ●
    │  ●    ●                    │  ╱ ●  ●
    │ ●  ●                       │╱ ●
    │●                    ●      │●
    └──────────→ x               └──────────→ x
  
  橡皮筋（拟合线）上下弹跳，
  最终停在"上面所有点到线的距离"和"下面所有点到线的距离"
  形成平衡的位置。
```

**橡皮筋的铁律：**

| 法则 | 直觉理解 | 数学含义 |
|------|---------|---------|
| **穿过重心** | 橡皮筋必然经过所有点的"重心"(x̄, ȳ) | 拟合直线一定过 (x̄, ȳ) |
| **两边绷力抵消** | 线上方的点到线的"铅垂距离"加起来 ≈ 线下方的点加起来 | 残差之和为零：Σ(y_i - ŷ_i) = 0 |
| **总绷力最小** | 橡皮筋追求"最松弛"的状态 | 残差平方和最小：min Σ(y_i - a·x_i - b)² |

> **一句话直觉：** 线性拟合是用一根拉直的线穿过散点群，让所有点到线的"铅垂距离"的平方和最小。不是随便画一根看起来差不多的线，而是"让所有点的冤枉感最小"的那根线。

### 动态想象：调节两根天线

```
  想象你要架设一根天线杆：
  
  天线杆的 slope（斜率 a）：决定倾斜角度
  天线杆的截距（b）：决定杆子插在哪里
  
  你有两个旋钮可以调：
  旋钮 A（调斜率）─→ 天线杆转角度
  旋钮 B（调截距）─→ 天线杆上下平移
  
  你的目标：让天线杆尽可能"靠近"所有接收点
  
  调节过程：
  1. 先大致让杆穿过点群中心
  2. 微调斜率，观察"总距离"（残差平方和）的变化
  3. 微调截距，观察"总距离"的变化
  4. 反复多次，直到两个旋钮都调到"最优"
```

**这就是最小二乘法的直觉：在斜率 a 和截距 b 的二维空间里，找那个让 Σ(y - ax - b)² 最小的 (a, b) 组合。**

### 反直觉警示：大脑会在哪里翻车

| 翻车点 | 直觉告诉你 | 实际真相 |
|--------|-----------|---------|
| **"肉眼拟合"不可靠** | "我画一根穿过最多点的线不就行了" | 目测的线和最优解可以相差很大。拟合是数学最优，不是视觉最优 |
| **铅垂距离 ≠ 垂直距离** | "距离就是点到线的垂线长度" | 回归用的是铅垂距离（y 方向的距离），不是垂直距离！因为 y 是"因变量"，我们关心 yn 的预测误差 |
| **过原点强制** | "y 和 x 成正比，所以 b=0" | 除非有物理依据（如胡克定律 F=kx），否则不要强制 b=0。自由拟合的线可能不经过原点 |
| **外推风险** | "有了线就能预测任何 x 的 y" | 拟合只在数据范围内有效。外推到数据范围外，结论可能完全荒谬 |

## 联结层（连什么）

### 跨学科锚点

**1. 物理：胡克定律实验**

```
  砝码质量（x）──→ 弹簧伸长量（y）

  理论上 y = (g/k)·x，经过原点的直线
  但实验测量有误差，数据点会散落在直线附近
  ↓ 线性拟合找出"最能解释这些带误差的数据"的直线
```

**2. 计算机：机器学习中的线性模型**

```python
# 最简单的机器学习模型：
from sklearn.linear_model import LinearRegression

# 给一堆 (x, y)，自动拟合最佳直线
model.fit(X, y)
y_pred = model.predict(X_new)

# 这背后和最小二乘法是同一个原理
```

**3. 工程：传感器校准**

```
  标准值（x）──→ 传感器读数（y）

  理想情况：y = x（经过原点的 45° 线）
  实际传感器有偏差，需要用多个标准点来拟合一条校准线
  然后拿这条线去修正所有测量值
```

### 向 Formal 的桥梁

**拆包后，落回哪条 formal 定理？**

| 高考场景 | 拆包结论 | Formal 着陆点 |
|---------|---------|--------------|
| 给定 (x,y) 数据求回归直线 | 找最优 (a,b) | 最小二乘法公式：a = Σ(x-x̄)(y-ȳ)/Σ(x-x̄)², b = ȳ - a·x̄ |
| 判断拟合是否合理 | 检查散点是否有线性趋势 | 散点图 + 相关系数 r |
| 比较两条拟合线 | 比较残差大小 | 误差平方和 SSE |
| 对变换后的数据做拟合 | 先做变换（如取对数）再拟合 | 可线性化的非线性回归 |

**首选工具**：最小二乘法公式（直接套公式求 a, b）
**次选工具**：散点图 + 目测趋势 + 相关系数验证

### 概念网络坐标

```
              ┌─────────────────┐
              │    成对数据      │
              │   (x_i, y_i)    │
              └────────┬────────┘
                       │
     ┌─────────────────┼─────────────────┐
     │                 │                 │
     ▼                 ▼                 ▼
┌─────────┐     ┌───────────┐     ┌─────────┐
│线性拟合  │     │ 曲线拟合   │     │ 独立性   │
│y=ax+b   │     │ y=f(x)    │     │ 检验     │
└────┬────┘     └───────────┘     └─────────┘
     │
     ▼
┌─────────┐
│最小二乘法│
│Σ(y-ŷ)²  │
└─────────┘
```

**拆包调用优先级：**
1. 看到散点有明显线性趋势 → 线性拟合
2. 线性趋势不明确 → 先算相关系数 r，|r| < 0.8 时警惕
3. 明显的曲线趋势 → 不要强行线性拟合，考虑变换或曲线拟合

### 历史/哲学视角：从天文到数据科学

```
18 世纪末（天文导航）：
  天文学家需要在多个含误差的观测值中找"最可能"的轨道
  拉普拉斯和高斯各自独立提出"使误差平方和最小"的原则
  ↓ 这就是最小二乘法——不是因为它"最直观"，而是因为它"最不容易被一个异常值绑架"
  
19 世纪末（遗传学）：
  高尔顿发现回归现象 → 回归直线进入统计学
  ↓ "回归"这个词来自生物学——向平均值回归
  
21 世纪（AI 时代）：
  线性回归是最简单的"机器学习"模型
  深度学习本质上也是"拟合"——只是不再拟合直线，而是拟合超曲面
```

## 迁移层（何时调用）

### 三类场景判定表

| 场景 | 识别信号 | 不拆包的代价 | 拆包收益 |
|:---|:---|:---|:---|
| 🎭 **包装** | "找出最能表示 y 和 x 关系的直线方程""求回归方程" | 没意识到这就是线性拟合，可能去列联表或做其他分析 | 3 秒锁定最小二乘法，直接套公式 |
| 🔀 **嵌套** | 题目先让你对数据做变换（取对数/开方），然后再拟合 | 不知道变换后还是拟合问题，卡在中间步骤 | 识别两步结构：变换→拟合，逐层处理，不混淆 |

### 拆包方向标（10 秒流程）

```
Step 1：识别结构类型（3 秒）
  • 看到 (x,y) 数据 + "求直线方程"？
    → 这是线性拟合，锁定最小二乘法
  • 看到"变换后求直线"？
    → 这是嵌套：先变换数据，再线性拟合

Step 2：剥外壳找骨骼（4 秒）
  • 画散点图（草图即可）确认线性趋势
  • 如果趋势不明显 → 先算相关系数 r 再决定是否拟合
  • 嵌套题：确认每步的输入是什么

Step 3：锁定 Formal 着陆点（3 秒）
  • 写下：拆包→【最小二乘法 / 线性回归】，立即套公式
```

### 向 Formal 切换铁律

> **拆包完成后，30 秒内必须切换为 formal 语言。**

| 场景 | 拆包结束的标志 | 切换后的 formal 动作 |
|:---|:---|:---|
| 包装题 | 识别出"求回归直线 = 求最优 (a,b)" | 套最小二乘法公式：a = S_{xy}/S_{xx}, b = ȳ - a·x̄ |
| 嵌套题 | 确认变换前后数据对应关系 | 先对每对数据做变换，再套公式拟合 |

---

### 直觉陷阱清单（仅拆包场景）

| 陷阱 | 何时失效 | 纠错方式 |
|------|---------|---------|
| **不管线性趋势强行拟合** | 数据明显弯曲还套直线公式 | 先画散点图！弯曲数据先尝试变换（取对数、开方） |
| **把铅垂距当垂直距** | 手工画线时自觉用"最短距离" | 回归分析用铅垂距离（y 方向），因为误差出现在 y 上 |
| **忽略过重心** | 算出的线不经过 (x̄, ȳ) 还觉得没问题 | 拟合线必须经过点 (x̄, ȳ)，用这个快速检验结果 |
| **外推滥用** | 用 x=100 带入拟合线，尽管数据范围是 [0,10] | 拟合线只在数据范围内可信，外推需额外论证 |
