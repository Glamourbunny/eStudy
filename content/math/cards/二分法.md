# 二分法

> 二分法是用"中点分割"逐步缩小根的范围的算法——每次把区间砍一半，直到区间足够小。

---

## 概念层

### 问题的起源

牛顿迭代需要计算导数，而且可能不收敛。二分法提供了一种更稳健的求根方法：

**前提条件**：f(x) 在 [a, b] 上连续，且 f(a)·f(b) < 0（异号）

**核心思想**：每次取中点，把区间分成两半，判断根在哪一半，然后继续。

> **一句话：** 二分法的本质是"不断取中点，锁定根的位置"——每一步都把搜索范围缩小一半。

---

### 本质定义（逻辑重构版）

**二分法算法：**

```
设 f(a)·f(b) < 0

Step 1: 取中点 c = (a + b)/2
Step 2: 计算 f(c)
Step 3: 判断：
        - 若 f(c) = 0，则 c 是根
        - 若 f(a)·f(c) < 0，则根在 [a, c]，令 b = c
        - 若 f(c)·f(b) < 0，则根在 [c, b]，令 a = c
Step 4: 重复，直到 |b − a| < ε
```

---

### 二分法 vs 牛顿迭代

| 对比项 | 二分法 | 牛顿迭代 |
|--------|-------|---------|
| 收敛速度 | 线性收敛（每步区间减半） | 平方收敛（接近根时极快） |
| 收敛条件 | 只需连续和异号 | 需要初始值在根附近 |
| 稳定性 | 稳定，保证收敛 | 可能发散 |
| 导数计算 | 不需要 | 需要计算导数 |

## 命题层

### 典型例题

**例题：用二分法求方程 x³ − x − 1 = 0 的根**

```
题目：已知方程在 [1, 2] 内有根，用二分法求近似解。
```

**深层分析：**

```
f(1) = 1 − 1 − 1 = −1 < 0
f(2) = 8 − 2 − 1 = 5 > 0

Step 1: c₁ = (1+2)/2 = 1.5，f(1.5) = 3.375 − 1.5 − 1 = 0.875 > 0
       根在 [1, 1.5]

Step 2: c₂ = (1+1.5)/2 = 1.25，f(1.25) = 1.953 − 1.25 − 1 = −0.297 < 0
       根在 [1.25, 1.5]

Step 3: c₃ = 1.375，f(1.375) ≈ 0.052 > 0
       根在 [1.25, 1.375]

继续迭代...
```

## 方法层

### 方法：二分法标准流程

**核心思想：** 不断取中点，缩小区间。

**标准操作流程：**

```
Step 1: 验证 f(a)·f(b) < 0
Step 2: 计算 c = (a + b)/2
Step 3: 计算 f(c)
Step 4: 根据 f(a)·f(c) 的符号更新区间
Step 5: 重复直到 |b − a| < ε
Step 6: 取 (a + b)/2 作为近似根
```

## 常见误区与纠正

| 编号 | 常见误区 | 错误根源 | 正确理解 |
|------|---------|---------|---------|
| 1 | "f(a)·f(b) > 0 就用二分法" | 未检查前提 | 必须异号才能保证有根 |
| 2 | "迭代次数算错" | 对数计算 | 区间每次缩小一半，n 次后区间长度为 (b−a)/2ⁿ |
| 3 | "精度判断错误" | 条件不清 | 是 |b−a| < ε，不是 |f(c)| < ε |
