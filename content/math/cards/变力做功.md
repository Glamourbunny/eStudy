# 变力做功

> **Intuitive 卡片不教你"怎么算"，而是教你"算什么"。** 变力做功是定积分最物理的直觉入口——力随位置变化时，功不再等于 F·s，而是每小段微功的累积。高考中它常被包装成"变力""非线性力"下求功的问题，或嵌套在能量守恒的综合题中。这张卡片让你 30 秒识别变力做功的积分本质。

---

## 视角层（像什么）

### 核心隐喻：逆风骑行的体力消耗

**变力做功 = 骑自行车顶风前行——风力随速度变化，你的体力消耗也在不断变化**

```
  匀速段：风力恒定，每公里消耗 50 卡路里
  加速段：风力加大，每公里消耗 80 卡路里
  上坡段：重力+风力，每公里消耗 120 卡路里
  
  总消耗 = 50×1 + 80×1.5 + 120×0.5 = 230 卡路里

  如果力连续变化（不是分段恒定）：
  总功 = ∫ F(x) dx —— 把每一寸的微功累加起来
```

**变力做功的三条直觉铁律：**

| 铁律 | 骑行类比 | 数学含义 |
|------|---------|---------|
| **力在变** | 不同路段需要的力气不一样 | F 不是常数，是位置 x 的函数 F(x) |
| **微功累加** | 把整段路切成无数小段，每段近似恒力 | dW = F(x)·dx，W = ∫ F(x) dx |
| **面积即功** | F-x 图上曲线下的面积 = 总功 | 定积分的几何意义就是累积量 |

> **一句话直觉：** 变力做功就是"不均匀的付出也要一笔笔记账"——力在变，就把路切成无穷小段，每小段按恒力算微功，全部加起来就是总功。

### 动态想象：弹簧的拉长过程

把变力做功想象成拉弹簧——越拉越费劲，但功是累积的：

```
  弹簧弹力 F = kx（胡克定律，力与伸长量成正比）

  x=0:  F=0    （初始，不用力）
  x=0.1: F=0.1k（稍用力）
  x=0.2: F=0.2k（更用力）
  ...
  x=L:   F=kL  （拉到最长，最费力）

  拉动过程中做的功：
    W = ∫₀ᴸ kx dx = ½kL²

  如果用力恒定的平均力 F_avg = kL/2，那么 W = F_avg × L = ½kL²
  巧合？不！是积分的结果。
```

**拉弹簧在高考里的翻版：**
- "力 f(x) 将物体从 x=a 移到 x=b，求功" → 直接套 W = ∫ₐᵇ f(x) dx
- "已知 F-x 图，求 0 到 L 的功" → 数曲线下格子（或积分）
- "电场力 F = kQq/x² 做功" → W = ∫ kQq/x² dx

### 反直觉警示：大脑会在哪里翻车

| 翻车点 | 直觉告诉你 | 实际真相 |
|--------|-----------|---------|
| **平均力陷阱** | "变力做功 = 平均力 × 位移，取起点和终点力的平均数就行" | **只有力随位置线性变化时（如弹簧）才等于算术平均！** 对非线性力（如 1/x²），平均力不等于 (F_start+F_end)/2 |
| **正功负功** | "力的方向和位移方向相反时，用绝对值就行" | **力和位移方向相同→正功；相反→负功。** 在整个路径上如果有时同时相反，必须分段积分 |
| **路径有关** | "变力做功只看起终点位置，跟走哪条路没关系" | **只有保守力（重力、弹力、电场力）才与路径无关！** 摩擦力做功与路径有关 |
| **积分上限** | "F-x 图面积就是功，不用管单位" | **坐标轴单位要一致！F(N) × x(m) = J，如果 x 是 cm 要先化 m** |

## 联结层（连什么）

### 跨学科锚点

**1. 物理：重力势能**

```
重力做功（恒力）：
  W = mgh（简单乘法）

弹簧弹力做功（线性变力）：
  W = ½kx²（积分 kx dx 的结果）

万有引力做功（非线性变力）：
  W = GMm(1/r₁ - 1/r₂)（积分 GMm/r² dr 的结果）

从乘法到线性积分到非线性积分——物理在课堂上带你走完了"定积分逐步升级"的全过程。
```

**2. 工程：吊车起吊**

```
吊车吊起重物时，钢丝绳张力随高度变化（因为钢丝自重）
非恒定力做功 → 需要用积分精确计算，决定电机功率选型。
少算一点都可能烧电机。
```

**3. 生活：按压打气筒**

```
打气筒越往下压越费劲（气体被压缩，压强增大）
F(x) = P(x)·A，P(x) 随 x 按 PV=常数变化
你做的功 = ∫ F(x) dx —— 不是手感能估出来的。
```

### 向 Formal 的桥梁

**拆包后，落回哪条 formal 定理？**

| 高考场景 | 拆包结论 | Formal 着陆点 |
|---------|---------|--------------|
| 已知 F(x) 表达式，求 W | 写出 W = ∫ F(x) dx，找原函数 | 微积分基本定理.md |
| 已知 F-x 图象，求功 | 面积 = 定积分，分段或数格 | 微积分基本定理.md + 分段处理.md |
| F 分段定义（如摩擦+推力） | 分段积分，正负分开 | 微积分基本定理.md + 分段处理.md |
| 能量守恒中含变力功 | 用动能定理替代积分：W = ΔE_k | 微积分基本定理.md |

**首选工具**：`定积分公式`——写出 ∫ₐᵇ F(x) dx，找原函数。

**次选工具**：能量守恒绕过积分——有时题目不要求直接积分，而是通过初末态能量差得出功。

**拆包调用优先级：**
1. 力是常数？→ W = F·s·cosθ，不需要积分
2. 力是位置的函数 F(x)？→ W = ∫ F(x) dx
3. 只知道初末状态？→ W_合 = ΔE_k，动能定理绕过积分

### 历史/哲学视角：功这个概念是工业革命的产物

"功"的概念在 19 世纪初才被科里奥利正式提出。之前的工程师只知道"力"和"距离"，不知道两者的乘积有如此深刻的能量意义。焦耳用实验证明了机械功和热的等价性——你做的功不会消失，只会转换成热能或其他形式的能量。这就是热力学第一定律。

变力做功的积分公式 W = ∫ F·dx，是把"能量转化"这一物理直觉的数学精确化：**无论力怎么变，功就是力在位移上的累积效果。**

## 迁移层（何时调用）

### 三类场景判定表

| 场景 | 识别信号 | 不拆包的代价 | 拆包收益 |
|:---|:---|:---|:---|
| 🎭 **包装** | "已知力随位置的变化关系 f(x)…""F-x 图象如图…""变力作用下…" | 用恒力公式 F·s 硬代，结果错误 | 识别变力 → 积分，恒力 → 乘法 |
| 🔀 **嵌套** | 力 F(x,v) 同时依赖位置和速度；多力联立做功 | 力的分解和叠加混乱 | 分别写出每个力的积分，按路径分段累加 |
| 🆕 **新定义** | "定义功密度 w(x)=…" | 新术语阻吓 | 功密度就是 dW/dx = F(x)，识别为力本身 |

### 拆包方向标（10 秒流程）

```
Step 1：识别结构类型（3 秒）
  • F 是常数？→ 恒力功 W = F·s·cosθ
  • F = F(x) 依赖位置？→ 变力，准备积分！
  • 力有正有负随位置变化？→ 分段积分！

Step 2：剥外壳找骨骼（4 秒）
  • 写出 F(x) 表达式或从 F-x 图读
  • 确定积分路径 [x_start, x_end]
  • 判断正功负功：力和位移同向？反向？

Step 3：锁定 Formal 着陆点（3 秒）
  • 问自己：直接积分还是用动能定理绕过？
  • 写下：拆包→【W = ∫F(x)dx / 动能定理】，立即切换严格语言
```

### 向 Formal 切换铁律

> **拆包完成后，30 秒内必须切换为 formal 语言。Intuitive 只负责定向，不负责计算。**

| 场景 | 拆包结束的标志 | 切换后的 formal 动作 |
|:---|:---|:---|
| 包装题 | F(x) 表达式已提取 | 写出积分，找原函数，代上下限 |
| 嵌套题 | 各力分解完成，路径分层确定 | 逐力逐段积分后求和 |
| 新定义题 | 新定义翻译为 F(x) 或 dW/dx | 用积分定义严格推导 |

**违反此铁律的后果**：积分上下限写反、正负功符号混乱、结果凭空差一个负号。

### 直觉陷阱清单（仅拆包场景）

| 陷阱 | 何时失效 | 纠错方式 |
|------|---------|---------|
| **平均力错觉** | F 非线性变化时用 (F₁+F₂)/2 估算功 | 除非力是位置的线性函数，否则老老实实积分 |
| **方向忽略** | 路程中有来回移动时仍给全程正功 | 画出 F 和 dx 的方向关系，反向段取负 |
| **路径依赖性** | 摩擦力做功只给一个数 | 摩擦力做功 = -f·s（路程，不是位移！） |
| **F-x 图正负混淆** | x 轴下方的面积当成正功 | 面积在 x 轴下方 = 负功，上方 = 正功 |

## 一句话总结

> **变力做功是"逆风骑行的体力账单"——力在变，就把路切成无穷小段，每段按恒力记账。高考里最坑的是"平均力幻觉"——非线性的力不能取平均数！F-x 曲线下的面积 = 功，x 轴下方面积是负功。30 秒判定恒力还是变力，变力就写积分表达式，交给 Formal 找原函数。**
