# 未知化已知

> **Intuitive 卡片不教你"怎么算"，而是教你"算什么"。** 考场上遇到没见过的题型怎么办？慌是没有用的——你要做的只有一件事：在这个陌生的题目里，找到你认识的那部分。

---

## 视角层（像什么）

### 核心隐喻：侦探破案——在陌生现场寻找熟悉的线索

**未知化已知 = 侦探进入一个全新的犯罪现场——环境是陌生的，但指纹、鞋印、弹道轨迹这些"线索"是最熟悉的工具**

```
  陌生现场                    熟悉的线索
┌────────────────┐        ┌────────────────┐
│ • 没见过的新定义│        │ • 这不就是求导？ │
│ • 奇怪的符号   │  提取  │ • 这是分类讨论  │
│ • 复杂的情境   │  ───→ │ • 相当于解方程  │
│ • 没听说过的定理│        │ • 本质是单调性  │
└────────────────┘        └────────────────┘

  现场是新的                  但线索是旧的
```

**侦探思维的三条铁律：**

| 铁律 | 侦探场景 | 违反的后果 |
|------|---------|-----------|
| **忽略外衣**：不看题目说了什么新词，看它让你做什么 | 不在意犯案手法叫什么花哨名字，直接看"用刀/下毒/…" | 被新名词吓住，忘了行为本质还是那个行为 |
| **功能性阅读**：每句话都在告诉你一个条件，"翻译"成你懂的语言 | 目击者说"看到一个黑影"→翻译为"嫌疑人身高约175，穿深色衣服" | 把题目当小说读，忘了每个词都有数学含义 |
| **类比搜索**：这个新东西最像学过的哪个？ | 新定义的运算 ⊕ 像加法/乘法/复合函数的哪个？ | 从零开始学新概念，而不是用旧概念"套"它 |

> **一句话直觉：** 未知化已知就是"在新的题目里认出旧的骨架"——别被新名词、新符号、新情境吓住，每句话都在告诉你一个数学条件，把陌生的帽子摘掉，底下的脸你一定见过。

### 动态想象：翻译官的工作流程

```
  题目原文（陌生语言）：
  "定义函数 f(x) 的弹性为 E(x) = x·f'(x)/f(x)，
   若 f(x) 的弹性为常数 c，求 f(x) 的表达式"

  翻译过程（未知→已知）：

  Step 1：抓定义
  "弹性" = 新名词，先别管它为什么叫"弹性"
  → 它的定义是 E(x) = x·f'(x)/f(x)

  Step 2：翻译条件
  "弹性为常数 c" = E(x) = c
  → x·f'(x)/f(x) = c
  → f'(x)/f(x) = c/x

  Step 3：识别熟悉结构
  f'(x)/f(x) = c/x
  → 左边是"导函数除以自身" = (ln|f(x)|)'
  → (ln|f(x)|)' = c/x
  → ln|f(x)| = c·ln|x| + C
  → f(x) = kx^c

  你不需要知道"弹性"在经济学里是什么——
  你只需要把它翻译成学过的方程！
```

### 反直觉警示

| 翻车点 | 直觉告诉你 | 实际真相 |
|--------|-----------|---------|
| **新定义题学新知识** | "需要理解这个新概念的本质" | 不需要"理解"，只需要"翻译"——把新定义代入条件，看看能推出什么 |
| **没见过 = 不会做** | "这题涉及的概念我没见过，放弃" | 新定义只是换了个名字——你只需要用它的定义替换新词 |
| **情境要读懂** | "物理/经济/生物的背景要搞懂才能做" | 背景知识只是包装——核心条件已经用数学语言写在下文里了 |

## 联结层（连什么）

### 跨学科锚点

**1. 化学：新物质的鉴定**
```
发现了一种未知化合物：
  → 测它的分子量（类比：确定函数类型）
  → 测它的官能团（类比：确定函数的特征行为）
  → 对比已知物质数据库（类比：找你学过的类似题型）
  → 鉴定完成："这是 XX 的同系物"（类比："这是 XX 题型的变体"）
```

**2. 生活：在新城市找路**
```
第一次到陌生城市：
  你不会重新发明"交通规则"——红绿灯、路标、导航
  你把陌生街道翻译成"直走、左转、右转"这些已知动作
```

### 向 Formal 的桥梁

| 未知场景 | 翻译为已知 | Formal 着陆点 |
|---------|-----------|--------------|
| "定义运算 a⊕b=…" | 把 ⊕ 代入条件，看它像加减乘除的哪个 | 四则运算.md |
| "定义新性质 P" | 把 P 的定义翻译成等式/不等式 | 构造函数.md, 方程转化.md |
| "动点的轨迹为 Γ" | 设 P(x,y)，代入条件 → 方程 | 解析几何.md |
| "增长率/弹性/半衰期"的新情境 | 把文字翻译成导数/比值/指数关系 | 微积分.md, 一阶ODE.md |

### 概念网络坐标

```
  陌生题目
      │
      ▼
  提取新定义 → 用符号写下来
      │
      ▼
  代入已知条件 → 消掉新定义，露出已知结构
      │
      ▼
  匹配学过的解法模板
      │
      ▼
  正常求解
```

## 迁移层（何时调用）

### 三类场景判定表

| 场景 | 识别信号 | 拆包收益 |
|:---|:---|:---|
| 🎭 **包装** | 新名词（弹性/散度/曲率…）+ 定义公式 | 5 秒替换新名词，剩下的就是标准计算 |
| 🔀 **嵌套** | 新定义 + 参数 + 条件多重叠 | 先翻译新定义，再处理嵌套结构 |
| 🆕 **新定义** | "定义…为…""若…则称…" | 这是典型的新定义题，核心任务就是翻译 |

### 拆包方向标（10 秒流程）

```
Step 1：圈出新定义（3 秒）
  • 题目里有没有没见过的词？
  • 圈出来，旁边写下它的数学定义

Step 2：代入条件翻译（4 秒）
  • 把新定义代入题目给的条件
  • 化简，看消掉了什么、剩下了什么

Step 3：匹配已知模板（3 秒）
  • 翻译后的表达式，是学过的哪种题型？
  • 如果是常规题型 → 用常规方法求解
  • 如果还不认识 → 继续拆
```

### 直觉陷阱清单

| 陷阱 | 何时失效 | 纠错方式 |
|------|---------|---------|
| **试图"理解"新概念** | 花时间理解新概念背后的"为什么" | 只翻译不深究——新概念只是缩写，用它替换后就不需要它了 |
| **翻译后不化简** | 代入新定义后的表达式很复杂就放弃了 | 化简！很多时候复杂表达式会魔术般地变成简单形式 |
| **类比错误** | 把新运算类比成错误的基本运算 | 代入具体数字测试：a=1,b=2 时计算结果是否支持你的类比？ |

## 一句话总结

> **未知化已知 = 考场上最重要的心理素质——别被新名词吓住。新定义不是新知识，只是旧知识的缩略语。学习新定义的方法是"代入→翻译→消去"，不是"理解→掌握→应用"。翻译完新名词后，底下的题目结构一定是你学过的。**
