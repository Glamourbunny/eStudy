# 杨辉三角

> **Intuitive 卡片不教你"怎么算"，而是教你"算什么"。** 杨辉三角在高考中常以"递推数表""组合恒等式"或"新定义三角阵列"的面貌出现。这张卡片只做一件事：让你在考场 30 秒内认出它是个"路径计数器"，把三角形上的数字换算成"有几条路能走到这个位置"。

---

## 视角层（像什么）

### 核心隐喻：弹珠游戏的落珠板

**杨辉三角 = 弹珠从顶端落下，经过层层钉子分流，最终落入底部格子的路径数分布**

```
将一个小球从三角形的顶端放下。球每次碰到一个钉子，随机向左或向右滚落。

              ●（起点：1 条路径到起点）
             / \
            ●   ●（第 1 层：1+1 种走法 = 左右各一条）
           / \ / \
          ●   ●   ●（第 2 层：到达中间的球？可以"左→右"或"右→左"= 2 条路径）
         / \ / \ / \
        ●   ●   ●   ●（第 3 层：1, 3, 3, 1）
       / \ / \ / \ / \
      ●   ●   ●   ●   ●（第 4 层：1, 4, 6, 4, 1）

  每个位置上的数字 = 球从顶端落到该位置的路径总数
```

**杨辉三角的三条铁律（弹珠版）：**

| 铁律 | 弹珠场景 | 数学对应 |
|------|---------|---------|
| **递推生成**：每个数 = 左上 + 右上 | 到达某钉子的球 = 来自左边上方的球 + 来自右边上方的球 | C(n,k) = C(n-1,k-1) + C(n-1,k) |
| **边界为 1**：最左列和最右列全是 1 | 一直向左滚（或一直向右滚）只有唯一的走法 | C(n,0) = C(n,n) = 1 |
| **对称性**：左半和右半对称 | 向左滚 k 次和向右滚 k 次，路径数一样 | C(n,k) = C(n,n-k) |

> **一句话直觉：** 杨辉三角就是"弹珠落点分布表"——每个数字告诉你"球有几种走法能落到这个位置"，而每一行的数字之和就是 2^n（因为每层分叉总数翻倍）。

### 动态想象：城市网格里的最短路径

```
出租车从 (0,0) 出发，只能向右或向上走，要到 (k, n-k) 有几条最短路径？

  ↑ ↑ ↑
  ↑ ↑ ●(2,1) → 从 (0,0) 到 (2,1)，需要向右 2 步+向上 1 步，共 C(3,2)=3 条路径
  ↑ ● ●
  ● ● ●
  (0,0) → → →

杨辉三角的第 n 行第 k 个数 = 从 (0,0) 走到 (k, n-k) 的最短路径数
```

**路径数"爆炸"的方式：**
- 第 10 行：1 + 10 + 45 + 120 + 210 + 252 + 210 + ... = 1024 = 2^10
- 第 20 行：总数 = 2^20 = 1,048,576 条路径
- 中考常出前 5-6 行，高考若遇到 10 行以上——直接上组合数公式，别硬画三角形

### 反直觉警示：大脑会在哪里翻车

| 翻车点 | 直觉告诉你 | 实际真相 |
|--------|-----------|---------|
| **中心最大** | "中间的数字最大，因为是两边路径汇聚的结果" | 对！第 n 行最大的是 C(n, ⌊n/2⌋)。中心数随 n 增大而爆炸性增长——第 100 行中心约有 1×10²⁹ |
| **行和 = 2^n** | "每行的和就是幂次翻倍" | 对，这是二项式定理取 a=b=1 的结果：(1+1)^n = Σ C(n,k) |
| **斐波那契斜线** | "杨辉三角和斐波那契有什么关系？" | 沿对角线求和：1, 1, 2, 3, 5, 8, 13… 正是斐波那契数列！这个隐藏关联不易察觉 |
| **二项式展开** | "杨辉三角就是 (a+b)^n 的系数表" | 对，但更重要的是：系数 C(n,k) 给了你展开式中 a^{n-k}·b^k 项的系数。高考常让你"求 x^5 的系数"——翻成杨辉三角语言就是"找路径落点" |

## 联结层（连什么）

### 跨学科锚点

**1. 概率论：高尔顿板**

```
高尔顿板 = 杨辉三角的物理实现

大量弹珠从顶端落下 → 每颗珠在每层随机向左或向右
↓
底部堆积成"钟形曲线"的轮廓
↓
这就是二项分布的直观演示：n 次独立试验中恰好成功 k 次的概率 ∝ C(n,k)
```

**2. 计算机：动态规划的状态转移**

```python
# 用杨辉三角递推计算组合数，O(n^2) 时间，O(n^2) 空间
triangle = [[1]]
for i in range(1, n+1):
    row = [1]
    for j in range(1, i):
        row.append(triangle[i-1][j-1] + triangle[i-1][j])
    row.append(1)
    triangle.append(row)

# C(n,k) = triangle[n][k]
```

**3. 生物学：细菌繁殖的谱系树**

```
一个细菌分裂繁殖：
第 0 代：1 个
第 1 代：2 个
第 2 代：4 个
...

如果追踪"某个基因恰好传了 k 代"的数量——就是杨辉三角！
```

### 向 Formal 的桥梁

**拆包后，落回哪条 formal 定理？**

| 高考场景 | 拆包结论 | Formal 着陆点 |
|---------|---------|--------------|
| 给出递推表，填数字 | 识别为杨辉三角变体 | 递推公式 a_{n,k} = a_{n-1,k-1} + a_{n-1,k} |
| "求 (x+2y)^8 中含 x^3 y^5 的系数" | 走 8 步中选 3 步为 x → C(8,3) | 二项式定理 + 系数组合 |
| "证明 ΣC(n,k)² = C(2n,n)" | 从 (0,0) 到 (2n,n) 分两段走 | 组合恒等式 → 范德蒙恒等式 |
| "新定义三角阵，递推规则为…" | 验证是否满足杨辉递推或其他自定义规则 | 递推生成法 + 通项公式推导 |

**首选工具**：`组合数公式`——C(n,k) = n!/(k!(n-k)!)，比画三角快。
**次选工具**：`递推填表`——当 n 较小时直接画三角，直观且不易错。

### 概念网络坐标

```
              ┌─────────────────┐
              │    组合数       │  ← 通项：C(n,k)
              │ （计数语言）    │
              └────────┬────────┘
                       │
                       ▼
              ┌─────────────────┐
              │   杨辉三角      │  ← 几何化：把组合数排成三角形
              │ （递推 + 可视化）│
              └────────┬────────┘
                       │
           ┌───────────┼───────────┐
           │           │           │
           ▼           ▼           ▼
     ┌─────────┐ ┌─────────┐ ┌─────────┐
     │二项式定理│ │组合恒等式│ │概率分布│
     │(a+b)^n展开│ │范德蒙等 │ │二项分布│
     └─────────┘ └─────────┘ └─────────┘
```

**拆包调用优先级：**
1. 看到"n 次选择中恰好选 k 次" → 组合数 C(n,k) → 杨辉三角定位
2. 看到"(a+b)^n 展开式第 k 项系数" → C(n,k) → 杨辉三角第 n 行第 k 列
3. 看到"递推数表" → 验证递推规则 → 判断是否是杨辉三角的变形

### 历史/哲学视角：一个三角形，三个文明

**杨辉三角是人类数学中最美的"重复发现"之一。**

```
11 世纪（中国·北宋）：贾宪《黄帝九章算经细草》
"贾宪三角"首次出现——用于开高次方（增乘开方法）。
↓ 中国的工程需要开方运算，三角作为"开方系数表"诞生。

13 世纪（中国·南宋）：杨辉《详解九章算法》
引用并推广了贾宪的工作。后世称"杨辉三角"。
↓ 在中国，三角是"计算工具"，不是"组合学对象"。

17 世纪（欧洲）：帕斯卡《论算术三角形》（1654）
独立发现，并系统研究了它的组合性质——与赌博问题结合。
↓ 在欧洲，三角是"概率论的工具"，开启了组合数学。

后世：命名为"帕斯卡三角"或"杨辉三角"，取决于你的文化语境。
```

> **哲学意义：** 同一个数学结构，在中国是"帮工程师开方的计算表"，在欧洲是"帮赌徒算概率的图"。杨辉三角告诉我们：**数学结构是超越文化的——不同的文明会用不同的方式发现同一个真理，但真理本身只有一个。**

## 迁移层（何时调用）

### 三类场景判定表

| 场景 | 识别信号 | 不拆包的代价 | 拆包收益 |
|:---|:---|:---|:---|
| 🎭 **包装** | "如图是一个数字三角形，每个数是它上方两数之和…" | 逐行硬算，算到第 8 行就眼花 | 3 秒认出这是"杨辉三角的包装"，直接用组合数公式 |
| 🔀 **嵌套** | "在杨辉三角中删去所有偶数，观察剩余图形的分形结构"（谢尔宾斯基三角） | 被"分形"概念吓住，不知如何下手 | 识别嵌套：杨辉三角(基底) + 模 2 运算(筛选层) → 谢尔宾斯基图案 |
| 🆕 **新定义** | "定义三角阵 T，其中 T(n,k) = T(n-1,k-1) × a + T(n-1,k) × b" | 被自定义的乘法系数(a,b)迷惑 | 识别这是"加权杨辉三角"——递推本质不变，只是权重变了 |

### 拆包方向标（10 秒流程）

```
Step 1：识别结构类型（3 秒）
  • 看到三角形数字排列表，每个数 = 左上 + 右上？
    → 拆"杨辉包装"：确认是标准杨辉三角还是加权变体

  • 看到"(a+b)^n + 求特定项系数"？
    → 拆"二项式包装"：系数的位置 = 杨辉三角第 n 行对应位置

  • 看到"定义新三角阵 T，满足递推规则…"？
    → 拆"新定义包装"：跑前三行，比较和标准杨辉三角的差异

Step 2：剥外壳找骨骼（4 秒）
  • 标准杨辉三角：直接上 C(n,k) = n!/(k!(n-k)!)
  • 加权杨辉三角：写出递推公式，可能需要推导新的通项
  • 行和/斜和问题：检查求和是否能化为已知恒等式

Step 3：锁定 Formal 着陆点（3 秒）
  • 问自己：递推公式 + 边界值，能否推出通项？
  • 写下：拆包→【组合数公式 / 递推求解 / 二项式定理】，立即切换严格语言
```

### 向 Formal 切换铁律

> **拆包完成后，30 秒内必须切换为 formal 语言。Intuitive 只负责定向，不负责计算。**

| 场景 | 拆包结束的标志 | 切换后的 formal 动作 |
|:---|:---|:---|
| 包装题 | 确认是标准杨辉三角 | 用 C(n,k) = n!/(k!(n-k)!) 直接算，不画三角形 |
| 嵌套题 | 层次关系已理清（杨辉层 + 筛选层） | 杨辉层按递推生成，筛选层用取模或除法处理 |
| 新定义题 | 新递推规则的三行样例已跑通 | 推导递推公式的通项，或证明与标准杨辉三角的关系 |

**违反此铁律的后果**：对大 n 继续手工填三角，效率低下且容易出错。

### 直觉陷阱清单（仅拆包场景）

| 陷阱 | 何时失效 | 纠错方式 |
|------|---------|---------|
| **第 0 行混淆** | 以为第 1 行是 (1,1)，忘记第 0 行是 (1) | 二项式展开 (a+b)^n 对应第 n 行，n 从 0 开始！(a+b)^2 的系数的第 3 行 |
| **组合对称性误用** | C(n,k)=C(n,n-k) 在"带权"的三角里不成立 | 加权后不一定对称——每次用对称性前先检查递推规则是否对称 |
| **斜线和漏项** | 斐波那契斜线求和时多算/漏算 | 画斜线时确认对角线方向：主对角线 = 1,1,1,…；副对角线 = 行中的项 |
| **大 n 手算** | n=20 时还坚持手写三角形 | n>10 直接上组合数公式，或编程/递推填表 |

## 一句话总结

> **杨辉三角是"弹珠落点图"——每个数字就是小球从顶端落到该位置的不同路径数。高考里它要么藏在二项式展开系数里，要么变为自定义递推三角阵，要么用来生成组合恒等式。你的任务：30 秒认出它是路径计数，翻成组合数公式 C(n,k)，然后用递推或阶乘去算。**
