# 七选五「语篇衔接」信号词库

> **模块定位**：七选五（10分）的核心技巧。不是靠翻译，而是靠信号词找到句间逻辑关系。

---

## 一、核心信号词分类

```
语篇衔接
    │
    ├── 指代衔接 ──→ it/they/this/these/such/one/another
    │
    ├── 逻辑衔接 ──→ however/therefore/besides/for example
    │
    ├── 词汇衔接 ──→ 同义替换/反义/上下义
    │
    └── 结构衔接 ──→ 段首总起/段中过渡/段尾总结
```

---

## 二、指代衔接（最常见）

| 信号词 | 指代内容 | 解题方法 |
|--------|----------|----------|
| **it** | 单数事物/抽象概念 | 前文找单数名词 |
| **they/them** | 复数人或物 | 前文找复数名词 |
| **this/that** | 前文提到的事/观点 | 前文找完整句子或观点 |
| **these/those** | 复数名词（近指/远指） | 前文找复数名词 |
| **such** | "这样的" + 名词 | 前文找同类描述 |
| **one...another...** | 同类事物中的不同个体 | 前文有复数概念 |

**解题技巧**：
```
看到选项中有指代词 → 回空格前文找被指代内容
看到空格后文有指代词 → 空格处必须有被指代内容

例：
前文：There are several ways to learn English.
选项：One of them is reading aloud every morning.
→ "them"指代"several ways"，衔接顺畅
```

---

## 三、逻辑衔接

| 逻辑关系 | 信号词 | 功能 |
|----------|--------|------|
| **递进** | besides, moreover, furthermore, in addition, also | 补充同类信息 |
| **转折** | however, but, yet, on the contrary, instead | 与前文相反 |
| **因果** | therefore, thus, as a result, so, consequently | 前文原因→本文结果 |
| **举例** | for example, for instance, such as, like | 具体化前文 |
| **强调** | in fact, actually, indeed, especially | 加强或修正前文 |
| **顺序** | first, second, then, finally, next | 步骤或列举 |

**解题技巧**：
```
转折信号 → 空格前后内容相反
因果信号 → 空格处是原因或结果
举例信号 → 空格处是概括性语句，后文是具体例子
顺序信号 → 空格前后有明确的步骤关系
```

---

## 四、词汇衔接

| 类型 | 说明 | 例子 |
|------|------|------|
| **同义替换** | 不同词表达同一概念 | important → significant → crucial |
| **反义** | 对立概念 | like → unlike / success → failure |
| **上下义** | 总概念与具体概念 | animal → dog / fruit → apple |
| **原词复现** | 同一词重复出现 | 最直接但也可能是陷阱 |

**解题技巧**：
```
空格前文出现"exercise" →
空格处可能出现同义词"workout""physical activity"

⚠️ 陷阱：原词复现不一定是答案
要同时考虑逻辑和结构是否匹配
```

---

## 五、结构衔接

| 位置 | 功能 | 常见表达 |
|------|------|----------|
| **段首** | 总起/主题句 | First of all, To begin with, The first point is... |
| **段中** | 过渡/递进 | Furthermore, Moreover, In addition, What's more |
| **段尾** | 总结/承上启下 | Therefore, In conclusion, To sum up, So... |

---

## 六、解题步骤（操作流程）

```
Step 1：看选项
  划出每个选项的关键词（指代词、逻辑词、核心名词）

Step 2：读文章
  重点看空格前后句子，划出关键词

Step 3：匹配
  ① 找指代对应（it/they/this指什么？）
  ② 找逻辑对应（转折？因果？递进？）
  ③ 找词汇对应（同义/反义/上下义）
  ④ 找结构对应（段首总起？段尾总结？）

Step 4：验证
  将选中的选项代入，通读检查是否流畅
```

## 七、高频陷阱（E1-E5）

```
┌─────────────────────────────────────────────────────────┐
│ 🔴 七选五常见陷阱                                         │
├─────────────────────────────────────────────────────────┤
│ E1. 只看原词复现就选定                                   │
│     原文和选项都有"exercise" ≠ 一定是答案                │
│     必须同时满足逻辑和结构匹配                           │
├─────────────────────────────────────────────────────────┤
│ E2. 忽视指代词导致逻辑断裂                               │
│     选项开头有"It is important..."                      │
│     但空格前文没有可以被it指代的单数名词                 │
│     → 此选项不匹配                                       │
├─────────────────────────────────────────────────────────┤
│ E3. 把段首空当作细节补充                                 │
│     段首空格通常是主题句，要概括全段                     │
│     不是承接上段的细节                                   │
├─────────────────────────────────────────────────────────┤
│ E4. 看到however就选转折                                  │
│     必须确认空格前后确实是对比/转折关系                  │
│     不能只看信号词，要看内容逻辑                         │
├─────────────────────────────────────────────────────────┤
│ E5. 代入后不复读验证                                     │
│     选完后必须通读全文，检查衔接是否自然                 │
│     常发现一个选项选了两次或漏选                         │
└─────────────────────────────────────────────────────────┘
```

## 八、自检清单

- [ ] 做题前先划出所有选项的关键词
- [ ] 优先考虑指代衔接（最直接）
- [ ] 段首空优先考虑主题句/总起句
- [ ] 不只看原词复现，还要看同义替换
- [ ] 选完后代入全文通读验证
