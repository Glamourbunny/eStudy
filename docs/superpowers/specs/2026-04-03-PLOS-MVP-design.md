# PL-OS MVP 设计文档

**版本**：V1.0
**日期**：2026-04-03
**项目**：高中个性化学习操作系统（PL-OS）
**核心定位**：培养终身学习者，而非应试工具
**MVP目标**：验证知识图谱可视化 + 无感化情绪监测的核心价值

---

## 一、项目定位与战略边界

### 1.1 核心定义

**不是**：学科内容推送工具、题库系统、单次选科咨询平台

**是**：基于教育心理学和学习科学的"认知操作系统"，提供：
- 情绪与动机调节的底层支持（Self-Determination Theory）
- 跨学科通用的元认知训练框架
- 可配置化的三年成长路径引擎

### 1.2 MVP核心价值

验证两个核心假设：
1. **知识图谱可视化**能否帮助学生清晰认知自己的知识缺口
2. **无感化情绪监测**能否在不打扰学生的情况下提供情绪支持

### 1.3 差异化价值

1. **情绪-认知双轮驱动**：学业情绪监测与干预先于知识推送
2. **元反思中枢**：将"学习过程本身"作为学习对象
3. **出题人眼光训练**：超越知识点记忆，培养模式识别能力（未来扩展）

## 四、核心功能流程

### 4.1 流程一：试卷上传 → 判卷 → 知识缺口分析

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  学生拍照   │───►│  OCR识别    │───►│  AI智能判卷 │───►│  显示判卷   │
│  上传试卷   │    │  题目+答案  │    │  (GPT-4o)  │    │  结果预览   │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
                                                              │
                                                              ▼
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  知识星系   │◄───│  节点着色   │◄───│  错题归因   │◄───│  学生校准   │
│  图可视化   │    │  🟢🟡🔴⚪   │    │  前置回溯   │    │  纠正误差   │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

**具体步骤：**

1. **学生拍照/上传试卷**
   - 支持拍照或相册选择
   - 图片格式：JPG/PNG

2. **OCR识别**
   - 提取题目文本 + 学生手写答案
   - 标注每项识别的置信度

3. **AI智能判卷 (GPT-4o)**
   - 直接接收图片，无需额外OCR步骤（GPT-4o原生多模态）
   - 输出结构化JSON：每题对/错 + 置信度 + 判断理由

4. **显示判卷预览**
   - 绿色 = 识别正确，红色 = 识别存疑
   - 让学生逐题确认

5. **学生校准**
   - 点错直接改
   - 确认后进入归因步骤

6. **错题归因到知识图谱**
   - 系统匹配错题 → 知识点（基于题目特征词/关键字）
   - 学生可手动调整关联

7. **前置知识回溯检查**
   - 若"三角函数"错，自动检查"单位圆"是否也🔴
   - 触发连通性预警

8. **知识星系图可视化**
   - 节点着色 + 点亮动画 + 预警高亮

### 4.2 流程二：无感化情绪监测

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  卷面分析   │    │  语音分析   │    │  情绪融合   │
│              │    │              │    │              │
│ • 涂改痕迹   │    │ • 语气语调   │    │ • 5维度评分  │
│ • 字迹潦草   │    │ • 关键词    │    │ • 情绪标签   │
│ • 做题节奏   │    │ • 停顿模式   │    │ • 分级干预   │
└─────────────┘    └─────────────┘    └─────────────┘
       │                │                   │
       └────────────────┼───────────────────┘
                        ▼
              ┌─────────────────┐
              │   情绪仪表盘     │
              │   5维度雷达图    │
              │   分级干预建议   │
              └─────────────────┘
```

**触发时机：**
- 学生主动上传试卷时自动触发（主要）
- 学生上传语音时自动触发（辅助）
- 学生可查看"是什么让我感到焦虑/厌倦"的解释

**情绪5维度（Pekrun学业情绪理论）：**
| 维度 | 说明 |
|------|------|
| 焦虑 | 考试/做题时的紧张程度 |
| 掌控感 | 对知识的掌握信心 |
| 意义感 | 学习目的和价值认同 |
| 疲劳 | 学习精力消耗 |
| 好奇 | 对新知识的探索欲望 |

**分级干预：**
| 级别 | 条件 | 干预方式 |
|------|------|----------|
| 轻度 | 某维度3-4分 | 认知重评话术（如"这道题87%同学首次会错"） |
| 中度 | 某维度2-3分 | 启动"情绪安全舱"（推送已掌握内容恢复掌控感） |
| 重度 | 某维度<2分 | 转人工心理辅导提示 |

### 4.3 流程三：知识图谱可视化

**页面布局：**
```
┌─────────────────────────────────────────────────────────┐
│  知识星系图          数学 · 三角函数          [筛选▼]   │
├─────────────────────────────────────────────────────────┤
│                                                         │
│        ○───○         ← 概念层（定义/公式）              │
│       /│\   \                                            │
│      ○ │ ○   ○      ← 程序层（解题步骤）                │
│     /  │   \ |                                           │
│    ●───●───●──●    ← ● = 错题节点（🔴）                │
│                                                         │
│  图例：🟢已掌握  🟡可用但费力  🔴碎片化  ⚪未激活    │
│                                                         │
├─────────────────────────────────────────────────────────┤
│  连通性预警：前置知识"单位圆"🔴⚪⚪ 阻断学习      │

```

**节点状态含义：**
| 状态 | 颜色 | 含义 |
|------|------|------|
| 🟢 | 绿色 | 自动化（能教他人/变式100%正确） |
| 🟡 | 黄色 | 可用但费力（能做对但耗时） |
| 🔴 | 红色 | 碎片化（时而对时而错） |
| ⚪ | 灰色 | 未激活（前置缺失） |

## 五、AI服务设计

### 5.1 GPT-4o判卷

**调用方式：** OpenAI API 直接发送图片

**Prompt设计：**
```
你是一位高中数学老师，请根据题目判断学生答案是否正确。
题目：[图片中的题目]
学生答案：[学生的手写答案]

请以JSON格式输出：
{
  "results": [
    {
      "question_id": 1,
      "is_correct": true/false,
      "confidence": 0.0-1.0,
      "reason": "判断理由"
    }
  ]
}
```

### 5.2 语音情绪分析

**调用流程：**
1. 语音上传 → 讯飞/腾讯API转文字
2. 文字内容 → 规则引擎分析
   - 关键词检测（"不会"/"太难了"/"听不懂"等）
   - 语气信号检测（停顿、重复、自我否定等）
3. 输出情绪标签

## 六、API设计

### 6.1 试卷相关

```
POST /api/exam/upload     # 上传试卷图片 → 返回OCR结果
POST /api/exam/judge      # GPT-4o判卷 → 返回判卷结果
POST /api/exam/confirm    # 学生校准确认 → 关联知识图谱
GET  /api/exam/:id        # 获取试卷详情
```

### 6.2 知识图谱相关

```
GET  /api/kg/nodes        # 获取知识图谱节点（可按学科/章节筛选）
GET  /api/kg/student/:id  # 获取学生个人知识图谱状态
PUT  /api/kg/node/:id     # 更新节点状态
POST /api/kg/mistake      # 关联错题到节点
```

### 6.3 情绪相关

```
POST /api/emotion/analyze        # 分析情绪（可基于exam或audio）
GET  /api/emotion/:student_id     # 获取学生情绪历史
GET  /api/emotion/:student_id/dashboard # 获取情绪仪表盘数据
```

### 6.4 语音相关

```
POST /api/audio/upload   # 上传语音 → 转文字 → 情绪分析
GET  /api/audio/:id      # 获取语音分析结果
```
