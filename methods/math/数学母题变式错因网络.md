# 数学母题变式错因网络

用三维坐标系定位错题，实现"错一道题 → 堵一类漏洞"的精准归因与跨章节变式联想。

---

## 一、为什么需要三维定位

传统错题本的局限：
- **线性排列** → 按时间/章节存储，三个月后无法检索
- **单题孤立** → 同源变式散落各处，无法形成题族
- **归因模糊** → 写"粗心""不会"等无效标签

数学版三维错题网络的核心升级：

| 维度 | 含义 | 定位示例 |
| --- | --- | --- |
| **X轴** | 数学对象 | 平面向量、指数函数、圆锥曲线 |
| **Y轴** | 知识结构 | 数量积运算、三角恒等变换、轨迹方程 |
| **Z轴** | 能力要求 | 运算求解、数形结合、逻辑推理 |

**三维定位的价值：** 当你找到"X=平面向量, Y=数量积运算, Z=运算求解"这个坐标，就精确锁定了知识网络中一个具体的坑位——下次遇到同类题，可直接索引；跨章节遇到相关题，可主动联想。

## 二、错题卡片模板

```markdown
---
name: 数学错题卡片
id: [母题编号]
date: [归档日期]
---

## 知识坐标

【X轴-对象】________（数/式/函数/方程/不等式/几何体/随机变量）
【Y轴-结构】________（概念/定理/运算规则/关系网络/思想方法）
【Z轴-能力】________（直观感知/逻辑推理/数学建模/运算求解/数据分析）

## 错因分类

- [ ] 概念误用（混淆了哪两个概念？）
- [ ] 公式记错（哪条公式？正确形式是什么？）
- [ ] 计算失误（哪一步？从何时开始算错？）
- [ ] 思路盲区（不知道往哪个方向想）
- [ ] 审题偏差（漏看了哪个条件？）
- [ ] 其他：________

## 题目信息

【题干】（精简到50字内，保留核心结构）

【关键条件】（导致错误/解题的关键条件，标注出来）

## 错因分析

【错因细化】（具体到哪一步、哪个符号、哪个隐含条件）
例：在利用平面向量数量积公式时，误将 |a||b|cosθ 写成 |a||b|sinθ，混淆了夹角与垂线的概念。

【正确思路】（用流程图或关键节点表示，非完整抄写）
例：向量垂直判定 → 点积为零 → 列方程求解

## 关联拓展

【变式联想】
- 与哪道已做题同源？（如：与第__题属于同一母题的变式）
- 可推广到什么情形？（如：从特殊角推广到任意角）
- 跨章节联想：这道题的解法能否迁移到__章节？

【回归教材】
对应教材：必修__ · 第__页 · 例__ / 习题__
定位：在教材中标注本题的"母题"位置

【下次复查】
复查日期：___________
是否已堵漏：□是 □否
堵漏方式：___________
```

## 三、每周执行动作

### 周五：归档

整理本周错题，填写标准化卡片：

1. **精炼题干**（50字内，保留核心结构）
2. **精准定位三维坐标**（X/Y/Z三轴）
3. **具体化错因**（不写"粗心"，写"哪一步、哪个符号、哪个隐含条件"）

### 周日：横向扫描

将本周错题按"错因"聚类，发现系统性漏洞：

**扫描清单：**

| 检查项 | 问题信号 | 行动 |
| --- | --- | --- |
| 错因集中度 | 某类错因占比>40% | 专项训练该类运算 |
| Y轴结构 | 定理适用边界频繁出错 | 回溯概念层卡片 |
| X轴关联 | 某对象连续出错 | 检查章节接口是否打通 |
| 变式遗漏 | 同源题反复出错 | 建立题族，标注变式链 |

### 月末：纵向穿透

1. 绘制错因分布柱状图
2. 标注"已堵漏"与"待解决"
3. 对反复出错的坐标点，单独制作专项训练卡组

## 四、数学版特色：跨章节变式联想

**联想路径图：**

```
平面向量（X轴）
    ├── 数量积（Y轴）→ 可变式到：三角函数求最值、解析几何垂直判定
    ├── 坐标运算 → 可变式到：空间向量、复数运算
    └── 线性表示 → 可变式到：线性方程组、矩阵初等变换
```

**跨章节联想训练：**

每道错题的"变式联想"栏，强制填写：

- 这道题的方法能否解__章节的哪类题？
- __章节的哪个知识点可以作为这道题的背景？

## 五、常见问题

**Q: 错题太多来不及整理怎么办？**
A: 优先处理高频考点错题（函数/导数/三角/向量/圆锥曲线/概率）。其他错题先拍照存档，周末批量处理。

**Q: 怎么快速确定三维坐标？**
A: 问自己三个问题：①这道题研究的是什么数学对象？②它考的是概念/定理/运算/关系中的哪一层？③解题需要什么能力？答案即坐标。

**Q: 错因"思路盲区"怎么修复？**
A: 回到对应章节的"方法层卡片"，强化该类问题的识别信号（如：看到"求最值+含参"→ 优先考虑函数法或不等式法）。
